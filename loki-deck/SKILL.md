# Loki Deck · 赛博小熊猫个人风格 PPT Skill

> 为 Loki 量身定制的 HTML PPT 生成系统。单文件输出，浏览器直接打开，键盘翻页，PDF 导出。

---

## 这个 Skill 做什么

生成一份**单文件 HTML** 的横向翻页 PPT，带有 Loki 个人品牌风格：

- **1920×1080 固定舞台 + scale 适配**（不在任何屏幕上 reflow）
- **12 套配色主题**（从正式到叙事到梦幻，覆盖全场景）
- **12 种版式**（封面/章节/列表/图文/数据/金句/对比/步骤/网格/结尾/杂志导读/引用插入）
- **IP 形象组件库**（9 种状态的小熊猫形象，可在任何版式中插入）
- **噪点纹理 + 光晕效果**（打破 AI 光滑感）
- **键盘翻页**（← → / Space / ESC 索引）
- **Noto Sans SC 900 标题 + Noto Serif SC 300 italic 副标 + Space Grotesk 数字 + JetBrains Mono 元信息**

---

## 何时使用

**合适**：
- 线下分享 / 行业内部讲话 / 公开演讲
- AI 工具评测 / 知识分享 / 观点表达
- 个人品牌内容输出
- 需要"一次做完，不用翻页工具"的网页版 slides

**不合适**：
- 大段表格数据（用常规 PPT）
- 培训课件（信息密度不够）
- 需要多人协作编辑（这是静态 HTML）

## 工作流（严格按顺序，不能跳步）

### Step 1 · 需求澄清 + 内容分析（动手前必做）

#### 1.1 · 如果用户给了主题或文章

**先分析内容，不要直接动手。** 基于内容做以下判断：

| 判断维度 | 如何判断 | 影响 |
|---------|---------|------|
| **内容主题** | 科技/生活/文化/情感/教育？ | 决定配色推荐 |
| **内容类型** | 观点分享/工具评测/教程/叙事？ | 决定版式组合 |
| **信息密度** | 大量文字/精简要点？ | 决定版式选择（大文字量版式 vs 精简版式） |
| **分享场合** | 公开演讲/内部分享/娱乐性？ | 决定配色和IP形象状态 |
| **预计页数** | 15分钟≈8-10页 / 30分钟≈15-20页 | 决定版式数量 |

#### 1.2 · 配色推荐逻辑

根据内容主题自动推荐配色（用户可改）：

| 内容主题 | 推荐配色 | 理由 |
|---------|---------|------|
| 科技/AI/工具评测 | 墨黑信号 / 电光蓝黄 / 灰蓝深度 | 科技感、冷调、专业 |
| 生活/心理/慢节奏 | 鼠尾草知识分子 / 摩卡慕斯 / 复古棕褐 | 温暖、人文、不刺眼 |
| 影评/文化/深度叙事 | 焦糖深蓝 / 酒红夜幕 / 黑色电影 | 深度、复古、情绪 |
| 演讲/正式/高端 | 午夜金色 / 墨黑信号 / 黑色电影 | 高级感、舞台感 |
| 创意/年轻/梦幻 | 梦幻紫调 / 摇滚墨水 / 墨黑信号 | 年轻、有冲击力 |
| 极简/知识/设计 | 摇滚墨水 / 鼠尾草知识分子 | 克制、高级 |

#### 1.3 · IP 形象推荐逻辑

根据内容场合选择 IP 形象状态：

| 场合 | IP 形象 | 文件名 |
|------|---------|--------|
| 封面/开场 | 原始3D形象（自信站姿） | `loki-original-3d.png` |
| 分享/演示 | 展示姿势（指向右上方） | `loki-presenting.png` |
| 写作/创作 | 写作状态（专注写字） | `loki-writing.png` |
| 科技/代码 | 兴奋+笔记本电脑 | `loki-excited-laptop.png` |
| 深度思考 | 思考状态（手托下巴） | `loki-thinking.png` |
| 成就/庆祝 | 庆祝姿势（双手举起） | `loki-celebrating.png` |
| 日常/轻松 | 酷+手机 | `loki-cool-phone.png` |
| 疲惫/加班 | 困倦+咖啡 | `loki-sleepy-coffee.png` |

IP 形象文件位置：`templates/ip-library/` 目录下，使用相对路径 `ip-library/loki-original-3d.png` 引用。

**使用方式**：在版式的图片预留位或装饰位插入 `<img>` 标签，用 `object-fit: contain` 适配。IP 形象适合放在封面右侧、结尾页中心、图文混排页的图片位。

### Step 2 · 生成方案 + 预览（Show Don't Tell）

**不要直接出最终版。** 先给用户一个方案预览：

1. **配色选择**：基于内容推荐 1 套配色，说明为什么
2. **版式规划**：列出每页用什么版式，说明节奏安排
3. **IP 形象选择**：说明在哪些页面放什么状态的 IP
4. **（可选）生成预览页缩略图**：让用户一眼看到整体节奏和风格

**预览页要求**：
- 所有页面以 960×540（50%缩放）排列展示
- 每页标注版式名称和配色
- 让用户一眼看到整体节奏和风格

**用户确认后才进入 Step 3。** 如果用户不 OK：
- 调整配色 → 换一套主题
- 调整版式 → 换同类型的另一个版式
- 调整 IP 形象 → 换另一个状态
- 调整排版 → 修改具体版式的参数

### Step 3 · 生成正式版

用户确认预览后，生成正式的单文件 HTML PPT：

1. **拷贝模板**：`cp <SKILL_ROOT>/templates/deck.html <项目目录>/index.html`
2. **选定配色**：从 `references/themes.md` 找到对应主题的 `:root` 块，替换模板中的 CSS 变量
3. **填充内容**：从 `references/layouts.md` 找到对应版式的 HTML 代码块，粘贴并修改文案
4. **插入 IP 形象**：在需要的位置插入 `<img src="...">` 
5. **检查类名**：确保所有 class 在模板的 `<style>` 里有定义
6. **检查节奏**：深浅交替，连续同色不超过 3 页

### Step 4 · 自检

生成后必须检查：

- [ ] 浏览器 Tab 标题不是占位符
- [ ] 字体全部加载（Noto Sans SC / Noto Serif SC / Space Grotesk / JetBrains Mono）
- [ ] 噪点纹理覆盖所有深底页面
- [ ] 页码标识正确（01/10, 02/10...）
- [ ] 品牌标识（赛博小熊猫 SVG）在每页右下角
- [ ] 进度条在顶部正确递增
- [ ] 键盘翻页正常工作（← → / Space / ESC）
- [ ] IP 形象正确显示（如果使用了）
- [ ] 没有文字溢出或重叠
- [ ] 深浅页面交替，没有连续 3 页同色

### Step 5 · 交付

- 单文件 HTML，浏览器直接打开
- 在回复中说明：用了哪套配色、哪些版式、IP 形象在哪些页面
- 提供文件路径

| 06 | 金句·居中引用 | 情绪锚点 | 核心金句 |

## 核心设计原则（10 条）

1. **所有颜色用 CSS 变量**：`var(--accent)` 不是 `#FF5E1A`。直接写 hex = 违规
2. **标题只用 Noto Sans SC 900**：不用得意黑做主标题（笔画多时糊）
3. **大数字用 Space Grotesk 200**：极细字重是辨识度的来源
4. **眉头标签全大写 + 0.2em 字间距**：品牌识别点
5. **噪点纹理必须覆盖深底**：打破 AI 光滑感
6. **深浅区块交替**：连续 3 页同色 = 视觉疲劳
7. **非对称布局优于对称**：有大小对比，不是等宽格
8. **字号要有极端反差**：标题大的时候副标极细极小
9. **IP 形象不是装饰**：放 IP 是为了品牌识别和情绪传达，不是空位填充
10. **不用 emoji**：任何输出不带 emoji

## 叙事弧模板（搭骨架用）

如果用户没有大纲，用这个模板搭：

```
钩子(Hook)       → 1 页   : 封面，抛一个反差/问题/硬数据
定调(Context)    → 1-2 页 : 章节过渡+背景说明
主体(Core)       → 3-5 页 : 核心内容，用要点/图文/数据/步骤穿插
转折(Shift)      → 1 页   : 金句/新观点
收束(Takeaway)   → 1-2 页 : 结尾CTA
```

## 迭代与优化

用户会不断发送新的参考素材（PPT截图/GitHub开源地址/别人的Skill），根据这些素材持续优化：

- 新版式 → 添加到 `references/layouts.md`
- 新配色 → 添加到 `references/themes.md`
- 新 IP 状态 → 生成后添加到 `ip-library/`
- 新组件 → 添加到对应版式或通用元素

每次优化后更新 `SKILL.md` 的组件数量统计。
