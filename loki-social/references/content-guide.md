# 原生小红书文案写作指南

## 核心原则

> **不要总结文章，要像一个人在跟朋友分享。**

所有之前尝试失败的根本原因：把公众号文章"翻译"成卡片。小红书用户要的不是信息摘要，是有人在跟他们说话。

---

## 不要把公众号文章转成卡片

❌ 错误做法：
1. 读公众号文章
2. 提取要点
3. 装进卡片 → 这就是 AI 味的来源

✅ 正确做法：
1. 理解选题的核心情绪和价值
2. 用小红书的口吻**重新讲一遍**
3. 每张卡片只讲一件事，用口语化的、有情绪的方式

---

## 小红书文案特征

### 语气
- 像在跟一个朋友发微信
- 带情绪：着急、心疼、忍不住、真的、绝了
- 允许不完整句式：「这个东西怎么说呢」
- 允许括号内心戏：「（不是画饼）」「（真能照着做的那种）」

### 句式
- 短句，但不要每个句号就断一次（那是 AI 特征）
- 自然的长短句交替
- 一段 2-4 行，不超过 5 行

### 词汇
- 用口语词：「压根」、「麻了」、「救命」、「谁懂啊」
- 用具体动词代替抽象名词：「做了」不是「构建」、「看不下去了」不是「发现问题」
- 禁止 AI 腔：「赋能」、「无缝」、「释放」、「下一代」、「解决方案」

### Emoji
- 每段 0-2 个，点缀而非堆砌
- 用于情绪表达（😮‍💨、✨）和视觉分隔（📍、📌）
- 不用来做标题装饰

### 数字
- 用汉字数字（一、二、三）而非阿拉伯数字（1、2、3）— 更有温度
- 结果数据用阿拉伯数字 + 符号：`200 → 800`

---

## 篇幅节奏

| 卡片 | 字数建议 | 情绪 |
|------|----------|------|
| 封面 | 20-40 字 | 钩子/好奇 |
| 大字报 | 15-30 字 | 冲击/共鸣 |
| 痛点共鸣 | 80-150 字 | 熟悉/认同 |
| 干货列表 | 60-120 字 | 信任/收藏 |
| 前后对比 | 30-60 字/侧 | 反差/震撼 |
| 对话气泡 | 50-100 字 | 亲和/行动 |
| CTA 收尾 | 40-80 字 | 温暖/号召 |

---

## 封面文案公式

```
[身份/场景] + [做了多久] + [情绪动词]

帮人改了一年简历
实在看不下去了
```

```
[数量] + [对象] + [反常识结论]

看了几千份简历
发现大部分人都在同一个点上吃亏
```

```
[成果] + [方法] + [号召]

用 AI 帮应届生找方向
免费 不用注册
拿去用
```

## SEO 要求

标题和前 100 字必须包含用户会搜索的关键词：
- 目标人群：应届生、校招、实习、找工作
- 场景词：简历、面试、求职、方向
- 品类词：AI 工具、免费工具、GitHub
- 情绪词：迷茫、不知道能干嘛、焦虑

---

## 发布笔记文案

除了卡片图片，还需要准备小红书正文（发帖时填写）：

**标题**（≤ 20 字，吸引眼球）：
- ✅ 「帮人改简历改麻了，做了个工具」
- ✅ 「应届生简历怎么写？试试这个」
- ❌ 「AI 求职教练使用指南」（太官方）

**正文**（发布时填写，可略长）：
- 第 1 段：重复封面钩子 + 个人身份（建立信任）
- 第 2 段：痛点共鸣（「很多同学明明做了很多……」）
- 第 3 段：引导看图（「左滑看具体怎么用」）
- 结尾：5-10 个 SEO 标签

**SEO 标签示例**：
```
#应届生求职 #简历怎么写 #AI工具推荐 #校招 #实习 #找工作 #GitHub开源 #免费工具 #应届生简历 #求职干货
```

## 自检清单

生成文案后逐项确认：

- [ ] 读起来像一个人在说话，还是像一篇公告？
- [ ] 有情绪词吗？（着急/心疼/实在/真的）
- [ ] 有括号内心戏吗？
- [ ] 每段 ≤ 4 行？
- [ ] 有 AI 腔吗？（赋能/无缝/释放/解决方案）
- [ ] 封面能通过缩略图测试吗？
- [ ] 前 100 字有搜索关键词吗？
