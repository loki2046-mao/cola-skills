# Rednote Copy Rules

The copy should feel native to Xiaohongshu while keeping the user's personal judgment.

## Core Rules

1. Keep the user's strongest original sentences when they sound human.
2. Do not write like a feature list unless the page is explicitly a feature proof.
3. Avoid generic AI phrases: "赋能", "提效神器", "一键生成", "智能化解决方案", "闭环能力" unless quoting.
4. Prefer concrete situations: "HR 转给业务负责人", "这活 AI 也能干", "负责/参与/协助".
5. Each page has one point. Delete "and also" content.
6. Put long install commands in caption or pinned comment, not main cards.
7. Use "我" when the author's viewpoint is part of credibility.
8. Do not use Chinese quotation marks in titles or big card copy unless quoting a real sentence from the source.
9. Avoid AI-template contrast sentences such as "不是 X，是 Y", "真正的 X 不是 Y", "你以为 X，其实 Y", and "别只 X，先 Y".
10. Avoid slogan-like transformation claims. Prefer concrete observations from the user, for example: "很多同学的简历不是没东西，是不会翻译" can become "很多同学有经历，但简历上只剩一句负责运营".
11. Do not overuse rhetorical questions. One question cover is allowed only when it sounds like a real user anxiety.
12. When adapting a WeChat article, keep the author's viewpoint but cut article scaffolding. Do not preserve section order just because the article used it.
13. The carousel copy should sound like a person making a useful note, not a brand launching a product.
14. If the source has a concrete example, use the example before the abstract lesson.

## Title Patterns

Use patterns like:

- Pain: `简历写得空，很多时候是信息没被翻译出来`
- Recruiter view: `我看学生简历时，会先看这几类证据`
- Transformation: `一段校园公众号经历，可以拆出这些能力`
- Proof: `这个 Skill 跑完后，会给到这些具体结果`
- How-to: `找实习前，先把这几类信息发给它`

Avoid title patterns like:

- `不是 X，是 Y`
- `你以为 X，其实 Y`
- titles with Chinese quotation marks
- vague AI-product words such as "神器", "保姆级", "闭环", "一键"
- fake urgency such as "我劝你一定要看完"
- over-promising such as "零经验直接上岸"

## Page Copy Density

- Cover: 1 main sentence + 1 support line + optional small tag.
- Pain/context page: 1 big claim + 2-3 concrete observations.
- Evidence page: screenshot/report/table + one explanation line.
- Prompt/how-to page: one short prompt artifact + field chips; full prompt goes to pinned comment if long.
- Closing page: one human thesis + one next action.

If a page needs more than 90 Chinese characters outside evidence screenshots, split it or move detail to caption.

## Caption Structure

Recommended:

1. Author viewpoint: why I care / why I made this.
2. Pain in 2-4 short paragraphs.
3. What the tool/Skill actually does.
4. One copyable prompt or use case.
5. CTA: install/comment/ask for example.

For WeChat-to-Rednote conversion:

- Image pages carry the hook, proof, and one clear action.
- Caption carries the origin story, caveats, and extra detail.
- Pinned comment carries long prompt templates, install paths, GitHub links, and commands.
- Do not paste the article intro into the caption. Rewrite it as short paragraphs.

## Pinned Comment Patterns

### Install Comment

Keep commands in comment if they are long. Use fenced code only in Markdown drafts, not inside image cards.

### Copy Prompt Comment

Provide one fill-in template:

```text
我是 [年级/身份]，[专业]，想找 [实习/校招/方向探索]。
我目前有 [课程项目/社团/兼职/比赛/自媒体/毕设/没有素材]。
我感兴趣的岗位是 [岗位名/JD/不知道]。
请帮我分析：我适合投什么、哪些写法太容易被 AI 替代、作品集怎么补、简历怎么写，以及接下来 7 天怎么做。
```
