# Xiaohongshu Style Modes

Use one primary style and up to two secondary styles. Style controls palette, type, layout density, components, and evidence handling.

## Global Visual Taste Rules

- Default background for student/career/AI/tool carousels should be bright and clean: optic white, porcelain white, mist white, cool silver, or a very light blue/green wash. Do not default to aged paper, pale yellow, beige, warm cream, or dusty orange.
- If a page starts to feel like "old paper", "brownish", "Claude beige", "earthy", "vintage scrapbook", or "low-end PowerPoint", revise the background before changing copy.
- Avoid pale yellow or beige as the dominant visual identity. Warm paper can be used only as a quiet surface, not as the whole style.
- Treat Pinterest/editorial references as layout systems, not color samples: strong crop, asymmetric grids, full-bleed evidence, dense/sparse page rhythm, and typography scale jumps matter more than decorative blocks.
- For career, student, AI, and tool topics, default away from tan, cream, muted orange, dusty burgundy, olive, and scrapbook palettes. These easily read as AI-template or low-end vintage.
- Prefer a clearer fashion/editorial base: optic white, soft silver, black ink, graphite, electric cobalt, tomato red, icy cyan, acid lime used in small doses, and one softer human accent such as blush or pale sky.
- Use color as a signal system: one page can be red-led, one can be blue-led, one can be mostly white, one can be dark. Do not fill every page with the same pastel wash.
- Avoid large bright color blocks that feel like AI-generated templates. Use restrained accents, material texture, photography/screenshot evidence, lines, and editorial rhythm instead.
- Avoid huge empty areas unless they create a deliberate magazine-style tension. Every empty area should balance typography, evidence, or image mass.
- Avoid generic rounded-card stacks. Use asymmetry, crop, overlap, editorial grids, narrow sidebars, evidence ledgers, and typographic contrast.
- Prefer premium palettes: porcelain white + graphite + electric cobalt, mist silver + deep navy + tomato red, ice blue + black ink, clean white + steel blue + tiny lime accent.
- Avoid earthy combinations such as beige + burgundy, tan + olive, coffee brown, archive yellow, and warm cream as the dominant palette unless the user explicitly asks for a vintage/archival feel.
- Do not make the whole carousel one color mood. Adjacent pages can share a system but should vary density, structure, and accent.
- Avoid quotation-mark poster tropes, fake handwritten stickers, soft blob gradients, pill-heavy layouts, and centered title-card pages unless the user explicitly wants a beginner-friendly worksheet look.
- Screenshots and proof blocks must be fully readable. Never crop midway through text. Crop only at natural UI boundaries, and add a small caption or ledger explaining what the viewer is seeing.

## Carousel Cohesion Gate

Before rendering the final PNGs, check the whole set as a deck:

- The pages must look like one carousel, not separate style explorations. Background, page chrome, typography scale, border treatment, and spacing rhythm should share one system.
- Do not switch between full black, full blue, full white, and grid paper backgrounds unless the user explicitly asks for a chaotic/maximalist set. Use accent panels instead of page-wide color jumps.
- Pick one dominant background family and keep it across at least 70% of pages. Let page variety come from composition, evidence format, and density, not unrelated palettes.
- Limit accent colors to two: one primary signal color and one risk/warning color. A third color is allowed only as a tiny UI detail.
- After export, inspect the pages together or as a contact sheet. If the first reaction is "these could be from different templates", revise before delivery.
- If the first reaction is "credible but dead", add controlled liveliness through crop, offset evidence windows, small black tags, light color slips, or stronger type scale. Do not add random stickers.

## S01 招聘评审桌风
- Use for: 简历、求职、实习、HR/业务负责人视角、岗位诊断
- Look: editorial hiring desk, sharp resume crops, audit marks, sidebar notes, proof ledger
- Palette: optic white `#FAFBFF`, ink `#111318`, cobalt `#2257FF`, tomato `#F0443E`, steel `#DDE4EE`
- Fonts: heavy sans for titles, mono for labels, occasional serif for concluding sentence
- Components: resume snippet, reviewer note, before-after, risk badge, evidence window
- Avoid: cute stickers, pure pastel, soft inspirational quotes, cheap folder-yellow or kraft-paper dominance

## S02 工具实测截图风
- Use for: AI 工具、Skill、软件、工作流、实际输出展示
- Look: app window, terminal-lite, report preview, screenshot crop, annotation, product evidence wall
- Palette: white/silver base + cobalt or cyan + tiny lime/red signal
- Fonts: sans + mono labels
- Components: app window, input/output split, screenshot ledger, feature proof, run result
- Avoid: fake screenshots without labeling; unreadable dense UI

## S03 明媚手账风
- Use for: 学生、轻知识、学习计划、生活化干货
- Look: neat notebook, soft color tabs, paper slips, mild hand note energy
- Palette: porcelain white, mist blue, muted sage, soft graphite, one restrained coral accent; no candy saturation
- Fonts: rounded sans or system sans, light mono tags
- Components: task cards, checklist, tiny calendar, sticky notes
- Avoid: overly cute stickers, childish doodles, clutter, low-end pastel blocks, yellow highlighter dominance

## S04 数据诊断风
- Use for: 评分、风险等级、测评、榜单、诊断报告
- Look: rating bars, score chips, quadrant, table fragments
- Palette: warm neutral + semantic green/yellow/red + one accent
- Fonts: sans + mono numbers
- Components: score strip, risk matrix, ranking rows, diagnosis summary
- Avoid: fake precision, unsupported percentages

## S05 红笔批注风
- Use for: 简历修改、改稿、避坑、错误/正确
- Look: marked-up draft, red/orange annotation, margin comments
- Palette: optic white, black ink, tomato red `#F0443E`, cool gray, optional cobalt
- Fonts: sans title, handwriting-like only for small notes if available
- Components: strike-through, margin note, correction stamp, before-after blocks
- Avoid: too many red marks causing anxiety

## S06 产品发布风
- Use for: 新 Skill、新工具、新功能、开源项目
- Look: product card, launch note, feature proof, clean CTA
- Palette: porcelain or graphite base + one brand color + functional second color
- Fonts: strong sans, mono metadata
- Components: feature proof grid, logo/type lockup, use cases, install CTA
- Avoid: corporate launch banner, empty feature list, big plain cream background

## S07 案例复盘风
- Use for: 项目复盘、真实案例、实践过程
- Look: case file, timeline, screenshots, problem/action/result blocks
- Palette: white/silver + graphite + one strong signal color; archive yellow only as a tiny note
- Fonts: editorial sans, serif for quotes
- Components: case timeline, decision note, proof gallery, lessons ledger
- Avoid: turning story into generic method

## S08 工作流拆解风
- Use for: SOP、教程、7 天计划、安装流程
- Look: rail, numbered cards, checklist, output standard
- Palette: white/silver + blue/green for steps + red or lime for current action
- Fonts: sans with clear hierarchy, mono step numbers
- Components: process rail, day-by-day plan, checklist, command box
- Avoid: putting long shell commands on main carousel unless tutorial-focused

## S09 观点海报风
- Use for: 强观点、反常识、认知刷新、争议标题
- Look: big type, sparse proof, strong one-sentence thesis
- Palette: can use strong red, cobalt, black, white, or silver. Avoid amber/orange unless the reference calls for it.
- Fonts: large heavy sans or editorial serif
- Components: thesis card, quote lens, contrast pair, one proof note
- Avoid: unsupported hot takes

## S10 深色工作台风
- Use for: 技术、代码、Agent、工程、debug、开发者
- Look: warm dark workspace, code/log window, amber signals
- Palette: dark base `#1A1816`, panel `#252220`, ink `#EBE4D6`, amber `#C87A45`, muted blue
- Fonts: mono for code, sans for explanation
- Components: code window, log excerpt, architecture chip, tool chain
- Avoid: neon cyberpunk, pure black, purple/blue gradients

## Selection Heuristics

- If the source contains "我工作里/我看过/我做过", preserve that viewpoint visually.
- If the source contains product output, make output visible by page 4 at the latest.
- If the source is a tool intro without proof, ask for or create a labeled demo page.
- If the topic is for students, use brighter palette but keep professional enough to save/share.
- If the topic is business/technical, prefer evidence-rich layouts over cute layouts.
