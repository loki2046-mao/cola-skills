# Loki Social · 赛博小熊猫个人风格社交图文 Skill

> 为 Loki 量身定制的社交图文生成系统。单文件 HTML，3:4 卡片，像素级切割截图。

---

## 这个 Skill 做什么

生成一份**单文件 HTML** 的小红书/小绿书图文系列：

- **3:4 卡片尺寸**（540×720 展示，1080×1440 实际渲染）
- **9 套配色主题**（覆盖全场景）
- **7 种版式**（X01-X07）
- **IP 形象组件库**（9 种状态，与 PPT skill 共用）
- **内容路由器**（先判断内容类型 → 选配色 → 选版式 → 出预览）
- **噪点纹理 + 光晕效果**
- **进度条**（10%→100% 给"通关感"）
- **像素级截图切割**（Chrome headless + 边界检测）
- **Noto Sans SC 900 标题 + Noto Serif SC 300 italic 副标 + Space Grotesk 数字**

---

## 何时使用

**合适**：
- 公众号文章转小红书图文
- 工具推荐 / 知识分享 / 观点表达
- 个人 IP 内容输出
- 小绿书（微信图文）内容

**不合适**：
- 视频内容（用视频工具）
- 纯文字（直接发）
- 需要动态交互（用 HTML 页面）

## 工作流（严格按顺序）

### Step 1 · 内容路由器（动手前必做）

**先分析内容，不要直接动手。** 内容路由器的核心逻辑：

```
用户给内容（文章/主题/大纲）
    ↓
判断内容类型
    ├─ 工具评测/推荐 → 配色：墨黑信号/电光蓝黄 → 版式：网格选择/图文混排/清单详情
    ├─ 知识分享/观点 → 配色：墨黑信号/灰蓝深度 → 版式：要点列表/金句/杂志导读
    ├─ 生活/个人IP → 配色：暖米咖啡/鼠尾草灰绿 → 版式：封面/图文混排/结尾
    ├─ 情感/叙事 → 配色：酒红夜幕/焦糖深蓝 → 版式：金句/引用插入/杂志导读
    └─ 娱乐/社交 → 配色：糖果粉紫/梦幻紫调 → 版式：网格选择/对比/步骤
    ↓
规划版式序列（封面→正文×N→结尾）
    ↓
选择IP形象状态
    ↓
生成预览 → 用户确认 → 正式版 → 截图切割
```

#### 内容类型 → 配色+版式映射

| 内容类型 | 推荐配色 | 推荐版式 | IP形象 |
|---------|---------|---------|--------|
| 工具评测 | 墨黑信号 / 电光蓝黄 | 网格选择 + 图文混排 + 清单详情 | 兴奋+电脑 |
| 知识分享 | 墨黑信号 / 灰蓝深度 | 要点列表 + 金句 + 数据大字 | 演示姿势 |
| 生活分享 | 暖米咖啡 / 鼠尾草知识分子 | 封面 + 图文混排 + 结尾 | 酷+手机 |
| 情感叙事 | 酒红夜幕 / 焦糖深蓝 | 金句 + 引用插入 + 杂志导读 | 思考 |
| 教程步骤 | 墨黑信号 / 电光蓝黄 | 步骤 + 要点列表 + 清单详情 | 写作 |
| 对比评测 | 摇滚墨水 / 墨黑信号 | 对比 + 网格选择 | 思考 |
| 娱乐社交 | 糖果粉紫 / 梦幻紫调 | 网格选择 + 对比 + 步骤 | 庆祝 |

### Step 2 · 规划版式序列

一篇小红书图文通常 6-10 张卡片。版式序列规划：

```
01 封面（居中大字，吸引点击）
02 章节过渡（如果是多章节内容）
03-08 正文（根据内容类型选 3-6 种版式交替）
09 金句（情绪锚点，适合转发）
10 结尾（CTA引导）
```

**版式交替规则**：
- 不要连续 2 张同类型版式
- 长文版式（X04/X05/X06）和精简版式（X02/X03）交替
- 深浅卡片交替（如果配色支持）

### Step 3 · 生成预览（Show Don't Tell）

**不要直接出最终版。** 先给用户一个预览：

1. **配色选择**：基于内容推荐 1 套配色
2. **版式序列**：列出每张卡片用什么版式
3. **IP 形象**：说明在哪些卡片放什么状态
4. **生成预览 HTML**：所有卡片以缩略图排列展示

用户确认后才进入 Step 4。

### Step 4 · 生成正式版

1. **拷贝模板**：`cp <SKILL_ROOT>/templates/carousel.html <项目目录>/index.html`
2. **选定配色**：从 `references/themes.md` 替换 CSS 变量
3. **填充内容**：从 `references/layouts.md` 粘贴版式代码块
4. **插入 IP 形象**：在需要的位置插入 `<img>`
5. **检查节奏**：版式交替、深浅交替

### Step 5 · 截图切割

生成正式 HTML 后，使用 Cola 的截图功能或系统截图工具进行截图切割：

```
# 截图SOP见 references/screenshot-sop.md
# 1. Chrome headless 全页截图（或用系统截图工具）
# 2. 像素级边界检测（扫描每行像素找色块边界）
# 3. 按边界切割成单独卡片图片
# 4. 输出到 output/ 目录
```

### Step 6 · 自检

- [ ] 所有字体已加载
- [ ] 噪点纹理覆盖深底卡片
- [ ] 进度条正确递增
- [ ] 品牌标识在每张卡片
- [ ] 版式 ≥ 5 种
- [ ] 没有文字溢出
- [ ] IP 形象状态匹配内容
- [ ] 截图切割边界精准

## 核心设计原则

1. **反AI味**：文字要填满画面，不要缩在角落
2. **大文字量**：信息密度要够，分块化排版
3. **视觉重心居中**：不要习惯性排左上方
4. **版式交替**：不要全是同一种版式
5. **噪点必须覆盖**：打破AI光滑感
6. **IP形象有目的**：放IP是为了品牌识别，不是填空
7. **进度条给通关感**：10%→100%
8. **不用emoji**
9. **三段式垂直分配**（必读 `references/layout-fill-principles.md`）：顶部标题区25%→中部主视觉区50%→底部辅助区25%，每段都有内容，禁止下半部空白
10. **装饰元素填空**：任何空白区域用几何图形/纹理/色块/小图标填充，没有纯空白
11. **IP形象是视觉主体**：占画面30-50%，不是小角标
12. **色块满版分段**：用满版色块切割画面，色块本身填满空间
13. **Horror Vacui**：每个角落都有东西，靠网格组织不杂乱

## 传播刀口论

图片承担 hook / proof / action，正文承担细节：

- **封面**（hook）：吸引点击，大字+IP
- **正文**（proof）：信息密度，分块化
- **金句**（hook×2）：情绪锚点，适合转发
- **结尾**（action）：CTA引导

## 迭代与优化

用户会不断发送新的参考素材（小红书截图/GitHub开源地址/别人的Skill），根据这些素材持续优化：

- 新版式 → 添加到 `references/layouts.md`
- 新配色 → 添加到 `references/themes.md`
- 新 IP 状态 → 生成后添加到 `ip-library/`
