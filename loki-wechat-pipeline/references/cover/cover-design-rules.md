# 公众号封面设计系统

为「赛博小熊猫Loki」公众号生成封面标题方案和21:9生图提示词。

## 触发条件

用户提供文章主旨、草稿、标题关键词，或直接说"帮我做封面"时调用。

## 输入协议

需要确认以下信息（缺失时主动追问）：

1. **文章主旨**：一句话说明文章核心内容
2. **文章类型**（可自动判定）：观点/教程/测评/实验/记录/观察
3. **核心卖点**：读者进来能拿到什么（免费工具？实操案例？认知刷新？）
4. **目标读者**：给谁看的（泛AI用户？教育工作者？内容创作者？）
5. **特殊要求**（可选）：指定IP姿势、指定配色偏好、指定标签

用户赶时间时，仅凭文章主旨即可启动全流程，其余参数用默认值并标注。

## 模块一：选题分类判定

### 判定规则

根据文章内容自动分为两类底色方案：

**深底方案（深赤铜渐变 + 白色/浅色文字）**
- 适用：观点文、思考文、深度分析、行业评论、价值观表达、现象解读
- 视觉气质：沉稳、有分量、有态度
- 渐变方向：从 #8a4528 到 #B8623C，带微妙暗角
- 文字颜色：纯白 #FFFFFF 或奶白 #faf3ec

**浅底方案（奶杏渐变 + 赤铜深色文字）**
- 适用：教程文、工具测评、操作指南、资源分享、干货集合
- 视觉气质：温暖、清爽、有获得感
- 渐变方向：从 #faf3ec 到 #f5e6d8 到 #eddccc
- 文字颜色：深赤铜 #8a4528 或正赤铜 #B8623C

### 判定输出格式

```
底色方案：深底 / 浅底
判定理由：一句话说明为什么选这个方案
```

## 模块二：标题生成

### 核心原则

Loki写标题最痛苦的点：想不出爆款标题。这个模块的目标是每次给出 3–5 个标题方案，覆盖不同传播场景，让Loki挑一个直接用。

### 标题生成规则

每个标题必须满足以下全部条件：

**1. 爆款前置**
把最有冲击力的信息放在标题最前面。优先使用：
- 具体数字（"花15分钟""测了30个""只要3步"）
- 痛点直击（"汇报被骂不饱和""背了就忘"）
- 反转/意外（"结果翻车了""我居然做出来了"）
- 悬念钩子（"被拿捏了半个月""有点不对劲"）
- 身份共鸣（"打工人""文科生""普通人"）

**2. 避开敏感词（硬约束）**
以下词汇不得出现在标题中，微信可能限流：
- 蒸馏、distill、distillation
- skill、agent、prompt
- AI助手（可用"AI""工具"替代）
- 大模型（可用具体产品名替代）
- token、API、fine-tune
- 其他明显技术黑话

**3. 干货获得感**
标题必须让读者预感到"进来有东西拿"：
- 明确交付物：免费工具、测试案例、实操教程、可复制方法、提示词
- 使用括号补充：（附链接）（免费开源）（可直接用）（保姆级）
- 暗示完整度：全流程、手把手、从0到1

**4. 转发欲**
标题本身是一个观点或洞察，转发到朋友圈不丢人：
- 能代表转发者的态度或品味
- 不油腻、不标题党、不廉价感叹
- 读完标题就想说"对，我也这么觉得"

**5. Loki语感**
- 口语化，有体温，像人说的话
- 不用"赋能/抓手/闭环/范式"等套话
- 不端着，不说教
- 不用「不是A而是B」句式

### 标题输出格式

每个方案包含：

```
方案 N：
主标题：xxxxxxxxxx
副标题：一句话点明干货或行动点
适用场景：（朋友圈转发 / 公众号首推 / 社群分享 / 搜一搜长尾）
亮点说明：一句话说明这个标题为什么可能有效
```

### 副标题规则

- 不超过20字
- 补充主标题没说完的信息：干货内容、行动点、身份标签
- 和主标题形成信息互补，不重复
- 示例：
  - 主标题提了工具 → 副标题说"免费开源，打开就能用"
  - 主标题提了观点 → 副标题说"附完整测试记录"
  - 主标题提了痛点 → 副标题说"3步搞定，亲测有效"

### 标签规则

生成 1–2 个短标签，放在封面角落位置：
- 不超过4个字
- 示例：思维训练、免费开源、AI工具、实操教程、深度好文、亲测分享
- 标签作用：快速传递文章属性，辅助视觉层次

## 模块三：封面构图系统

### 基本规格

- 比例：21:9（如 2100×900px）
- 风格：3D渲染质感 + 科技感 + 温暖色调
- 必须有设计感，不能像PPT

### 构图分区

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   [标签]                                         │
│                                                  │
│   ██ 主标题（大/粗/醒目）██                       │
│   副标题（小/细/补充信息）                         │
│                              🐼 赛博小熊猫Loki   │
│                              + 主题互动元素       │
│                                                  │
│   ←───── 文字区 60% ──────→←─── IP区 40% ───→   │
└──────────────────────────────────────────────────┘
```

**文字区（左侧约60%）**
- 主标题：大号粗体，视觉重心，占文字区主要面积
- 副标题：小号细体，在主标题下方，提供补充信息
- 标签：小号，放在左上角或主标题上方，带圆角背景色块
- 文字排列有明确层级：标签 → 主标题 → 副标题，大小对比明显

**IP区（右侧约40%）**
- 赛博小熊猫Loki必须出现
- IP核心特征：3D风格、穿黑色赛博机甲、胸前发光"LOKI"字样、圆润可爱但有科技感
- IP必须和文章主题产生互动（详见下方互动模板）

**背景**
- 禁止纯色平涂
- 必须有渐变 + 至少一种微妙纹理（噪点/几何线条/光斑/粒子/网格）
- 要有"呼吸感"：不能太满太密，留出视觉空间
- 深底方案：深赤铜渐变底 + 暗色纹理 + 微光效
- 浅底方案：奶杏渐变底 + 浅色纹理 + 柔光效

### 字体指导

- 主标题：效果等同思源黑体 Heavy / 粗黑体，笔画饱满，视觉冲击力强
- 副标题：效果等同思源黑体 Regular / 细体，与主标题形成粗细对比
- 标签：小号，带背景色块，圆角矩形
- 中文文字必须清晰可读、无错字、无乱码

### IP互动模板

根据文章类型选择合适的IP姿势和互动元素：

| 文章类型 | IP姿势 | 互动元素 |
|---------|--------|---------|
| 工具测评 | 双手捧着/展示 | 发光的工具界面、悬浮屏幕 |
| 教程/指南 | 指向某个方向、竖大拇指 | 步骤箭头、清单板、齿轮 |
| 思考/观点 | 托腮思考、盘腿坐 | 发光大脑、思维气泡、星空 |
| 实验/测试 | 拿着放大镜、操作台前 | 试管、数据面板、对比图 |
| 资源分享 | 打开宝箱、递出礼盒 | 宝箱、文件夹、下载图标 |
| 开发记录 | 坐在电脑前、敲键盘 | 代码窗口、火箭、进度条 |
| 行业观察 | 站在高处远眺、拿望远镜 | 远景城市、信息流、时间线 |
| 辩论/对比 | 双手摊开权衡、天平姿势 | 天平、VS标志、对比面板 |

### 色彩硬约束

**主色系**
- 赤铜色：#B8623C（核心品牌色）
- 深赤铜：#8a4528（深底渐变暗面）
- 浅赤铜：#c97a52（高光/过渡）

**辅色系**
- 奶杏底渐变：#faf3ec → #f5e6d8 → #eddccc

**禁止撞色**
以下颜色不得作为主要配色出现：
- 浅绿
- 深蓝
- 橙橘
- 紫色

**点缀色**（仅用于小面积光效/发光元素）
- 暖白光：用于IP机甲发光、文字光效
- 淡金色：用于高光点缀
- 赤铜色本身的高亮版本

## 模块四：提示词输出

### 提示词结构

输出完整的英文生图提示词，包含以下所有要素：

```
1. 画面比例和质量声明
2. 整体风格和渲染方式
3. 背景描述（渐变方向、纹理类型、光效）
4. IP角色描述（外观、姿势、互动动作）
5. 互动元素描述（与文章主题相关的道具/场景元素）
6. 文字内容（主标题、副标题、标签的具体内容和排版位置）
7. 色彩方案（具体hex值）
8. 质感和细节（材质、光影、景深）
9. 负面约束（要避免的东西）
```

### 提示词模板

```
[Aspect ratio and quality]
Ultra-wide 21:9 banner, 2100x900px, 4K resolution, sharp and crisp details.

[Overall style]
3D rendered digital illustration with a polished, modern tech-warm aesthetic. Magazine-quality cover design with clear visual hierarchy.

[Background]
{深底/浅底对应描述}. Subtle {纹理类型} texture overlay. {光效描述}. The background has a breathing, spacious feel — not cluttered.

[IP Character — right 40% of frame]
A cute 3D cyber red panda character wearing black cyber-mech armor with a glowing "LOKI" text on the chest plate. The character is {姿势描述}, {互动动作描述}. The character is rounded, adorable yet techy, with warm ambient lighting on the armor edges.

[Interactive elements]
{与主题相关的互动元素，具体描述}

[Typography — left 60% of frame]
Main title in bold Chinese heavy sans-serif font (similar to Source Han Sans Heavy): "{主标题中文}"
Subtitle in lighter weight below: "{副标题中文}"
Small tag label with rounded rectangle background in the upper-left area: "{标签}"
All Chinese text must be perfectly legible, sharp, with no artifacts or garbled characters.
Clear typographic hierarchy: tag (small) → title (large/bold) → subtitle (medium/light).

[Color palette]
{对应色彩方案，含具体hex值}

[Texture and finish]
{材质、光影、景深描述}

[Negative prompt]
No watermarks, no blurry text, no garbled characters, no extra limbs, no neon green, no deep blue, no orange, no purple as dominant colors, no flat solid backgrounds, no clipart style, no low-resolution elements.
```

## 执行流程

收到文章信息后，严格按以下顺序执行：

### Step 1. 分析文章

- 提取核心主旨、卖点、目标读者
- 判定文章类型

### Step 2. 选题分类判定

- 输出：深底 or 浅底 + 判定理由

### Step 3. 生成标题方案

- 输出 3–5 个标题方案
- 每个含：主标题、副标题、适用场景、亮点说明
- 同时输出 1–2 个标签

### Step 4. 确定构图方案

- 选择IP姿势和互动元素
- 确定背景纹理和光效方案
- 说明构图逻辑

### Step 5. 输出完整提示词

- 基于选定的标题方案，输出完整英文生图提示词
- 如果用户没有选定标题，默认用第一个方案生成提示词
- 同时标注：如果换了其他标题方案，提示词中哪些部分需要替换

## 质量自检清单

输出前逐项检查：

- [ ] 标题是否口语化、有体温、像人说的话？
- [ ] 标题是否避开了所有敏感词？
- [ ] 标题是否用了「不是A而是B」句式？（禁止）
- [ ] 标题最前面是否放了最有冲击力的信息？
- [ ] 副标题是否和主标题信息互补？
- [ ] 标签是否简短（≤4字）且准确？
- [ ] 提示词是否包含了完整的构图要素？
- [ ] 色彩是否在品牌色系内、没有撞色？
- [ ] IP描述是否包含黑色赛博机甲 + 发光LOKI字样？
- [ ] 背景是否有渐变 + 纹理（不是纯色）？
- [ ] 中文文字是否要求了清晰可读？
- [ ] 提示词是否包含21:9比例和4K分辨率？

---

## 快捷用法

用户可以直接说：

- "帮我做封面，文章是关于xxx的" → 全流程执行
- "只要标题方案" → 只执行 Step 1–3
- "只要提示词" → 用户提供标题，直接执行 Step 4–5
- "换个标题重新出提示词" → 替换标题部分重新输出

---

## 注意事项

1. 标题是核心交付物，质量要求最高。宁可多花时间打磨标题，不要为了快而出平庸方案。
2. 提示词要具体到可以直接丢进生图模型，不要出现含糊描述。
3. IP形象描述每次都要完整写，不要省略——生图模型没有记忆。
4. 色彩约束是硬性的，不要为了"好看"而引入撞色。
5. 如果用户的文章主旨不够清晰，先追问再动手，不要猜。
