# Article Type Patterns

This file describes Loki's main article categories, along with the matching state, logic, structure, and sentence movement.

## 1. 模型/产品测评

### Common topics

- 大模型更新
- AI 工具新功能
- 产品值不值得试
- 某个技术争议值不值得认真看

### Writer state

- 快速响应热点，但不盲目追热
- 先被事件触发，再转向自己的判断
- 有明显的"我先替你测 / 我先替你捋清楚"姿态

### Typical logic

1. 先抛事件或争议
2. 指出大家真正没搞懂的点
3. 用类比把技术问题讲人话
4. 给出带条件的判断
5. 说明适合谁、不适合谁

### Structural habits

- 开头常带热度感、时效感、评论区氛围
- 中段用例子、比喻、假设场景降低门槛
- 结尾一般不会喊口号，而是收束到实际意义

### Style cues

- 语气更快
- 带判断，但不装权威
- 经常用 `其实`、`但我发现`、`说白了`

## 2. 图像生成 / 案例实验

### Common topics

- 生图模型实测
- 提示词玩法
- 风格实验
- 连续图片案例

### Writer state

- 测试者 + 展示者
- 兴奋感更强，但仍然保留筛选意识
- 喜欢直接展示结果，再倒推方法

### Typical logic

1. 用一句情绪化判断破题
2. 直接告诉读者这次测了什么
3. 放结果、玩法、结论
4. 再补限制条件和坑点
5. 告诉读者怎么拿去用

### Structural habits

- 很多文章会先给一句总评
- 允许"全是图""先放图""提示词在后面"这类直接表达
- 对图片型内容，正文常短，节奏更偏展示

### Style cues

- 更口语、更轻快
- 偶尔会有夸张感，但通常是可控的夸张
- 常出现 `上图`、`直接看`、`拉到最后`

## 3. 效率方法 / 工作流

### Common topics

- 工作流推荐
- 办公提效
- 工具替代方案
- 零门槛方案

### Writer state

- 非常强的用户同理心
- 会主动考虑"普通人用不用得上"
- 很看重门槛、兼容性、是否需要魔法、是否开箱即用

### Typical logic

1. 从留言、评论区、现实工作痛点切入
2. 指出理想工具和真实用户之间的 gap
3. 给低门槛替代方案
4. 强调真实可用性，而不是最先进
5. 最后再讲具体步骤或提示词

### Structural habits

- 经常把"能用 > 好用 > 完美"这类价值排序说出来
- 喜欢把复杂问题拆成普通人能执行的动作
- 语气比测评更照顾读者焦虑

### Style cues

- `说实话`、`我突然意识到`、`你推荐的东西再牛，用户用不上，就是零`
- 会讲筛选过程，说明自己不是随便推荐

## 4. 教育实践

### Common topics

- AI + K12
- 教学卡片、课堂材料、模板
- 儿童英语、语文、古诗、阅读

### Writer state

- 明显的落地导向
- 很关注学生、老师、家长的实际处境
- 不把 AI 当炫技，而是当教学工具

### Typical logic

1. 从教育场景的真实困难切入
2. 指出传统方式的卡点
3. 展示 AI 如何帮忙
4. 强调可打印、可上课、可直接使用
5. 给读者后续获取方式或互动入口

### Structural habits

- 会写"我一直在做更落地的事情"
- 对使用对象非常明确：老师、家长、孩子
- 更强调帮助感，而不是技术优越感

### Style cues

- 语言更柔和
- 强调 `图文并茂`、`结构清晰`、`直接打印就能用`

## 5. 产品 / 开发记录

### Common topics

- 自己做的网站、小程序、工具
- 无代码、轻开发、AI 协作开发
- 从内容走向产品的个人转变

### Writer state

- 带明显个人叙事
- 会自嘲，也会诚实暴露不懂技术的部分
- 重点不是炫开发能力，而是表达 "我居然也做出来了"

### Typical logic

1. 先承认身份反差或起因
2. 快速给出结果
3. 解释为什么会做、怎么做出来
4. 拉近与同类读者的距离
5. 给出鼓励式的行动建议

### Structural habits

- 经常用 `文科生`、`不会编程`、`没写一行代码`
- 用故事推进，不用规格说明推进
- 更有个人成长感

### Style cues

- `事情是这样的`
- `不装了`
- `不是，我开玩笑的……`

## 6. AI观察 / 观点评论

### Common topics

- AI 热点的祛魅
- 技术与普通人生活的距离
- 行业趋势背后的真实问题
- 内容创作和工作方式的变化

### Writer state

- 站在热闹之外观察
- 比起说"这很新"，更关心"这有什么实际意义"
- 带一点阶段性的自我反思

### Typical logic

1. 从自己最近的感受写起
2. 把个人变化转成行业观察
3. 提出一个更底层的问题
4. 去除神话和噪音
5. 落回普通人的实际处境

### Structural habits

- 经常先讲自己的状态变化
- 不是上来就站队，而是重新定义问题
- 常用一句问题做中轴：`这东西到底帮谁解决了什么？`

### Style cues

- `我好像不怎么追热点了`
- `什么叫祛魅？`
- `对大部分普通人来说`

## Cross-Type Constants

No matter which type Loki is writing, these stay stable:

- always anchored in lived context
- always trying to lower comprehension threshold
- always judging usefulness, not only novelty
- always writing to real readers, not imaginary industry insiders
- always leaving the reader with a next thought, next step, or next question
