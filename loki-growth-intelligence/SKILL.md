# Loki Growth Intelligence · 增长情报引擎

> 你不缺生产工具，缺的是"做什么才是对的"的判断依据。这个 Skill 把散落在各处的数据变成决策信号。

## 四个模块

### 模块 A · 选题情报

从公众号历史数据中提取选题规律，结合实时热点，给出有数据支撑的选题建议。

**工作流：**
1. 读取公众号历史文章数据（飞书多维表格 / wechat_article_metrics.json）
2. 按 4 个维度分类每篇文章：
   - **选题类型**：工具测评 / 教程实操 / 行业观察 / 个人体验 / 选题迁移 / 纯观点
   - **内容形式**：长文 / 图文 / 短文
   - **发布时段**：工作日 vs 周末 / 深夜 vs 白天
   - **是否商单**：是 / 否
3. 计算每类的**均值表现**：阅读 / 转发 / 关注转化 / 互动率
4. 标记**超预期文章**（高于均值 1.5σ）和**扑街文章**（低于均值 0.5σ）
5. 提炼规律：哪种选题类型 × 哪种内容形式 = 最高投产比
6. 扫描 24h 实时热点（AI 领域），与历史规律交叉匹配
7. 输出 **3 个选题建议**，每个包含：
   - 选题方向 + 预估表现区间
   - 为什么推荐（数据依据 + 热点依据）
   - 参考历史爆款（具体文章名 + 数据）
   - 建议内容形式和篇幅

**输出格式：**
```md
📊 选题情报 | YYYY-MM-DD

🔥 Top 3 选题建议

1. [选题方向]
   📈 预估阅读：2000-3500（基于同类均值 1800）
   💡 推荐理由：近7天热点 #XXX 讨论↑，你的历史同类选题均值阅读 2200
   📝 建议形式：长文 2500字，教程+个人体验混搭
   📎 参考爆款：《XXX》（阅读 5200，转发 89）

2. ...

📋 本周选题池状态：已排期 X 篇 / 空位 X 篇 / 商单浓度 X%
⚠️ 数据不足：你的"纯观点"类样本仅 8 篇，统计意义有限
```

### 模块 B · 投产比追踪

追踪每个 AI 订阅的用量和产出，算出钱花在了哪里、哪些该续费、哪些该砍。

**工作流：**
1. 读取飞书「AI 积分追踪」表格
2. 按月统计每个订阅的：
   - 费用（月费 / 年费折算月费）
   - 实际使用次数 / 积分消耗
   - 关联产出（这个工具产出了什么内容/商单/粉丝增长）
3. 计算每个订阅的**单次使用成本**和**月投产比**
4. 标记：高频高产出（续）、低频高成本（砍）、高频低成本（超值）
5. 输出月度投产比报告

**输出格式：**
```md
💰 AI 订阅投产比 | YYYY年MM月

| 订阅 | 月费 | 使用次数 | 单次成本 | 关联产出 | 判定 |
|------|------|---------|---------|---------|------|
| Cola | ¥XX | XX次 | ¥X.X | 公众号X篇+小红书X张 | ✅ 续费 |
| ListenHub | ¥XX | XX次 | ¥X.X | X期播客+X个TTS | ⚠️ 观望 |
| ... | | | | | |

🎯 月度总投入：¥XXX
📈 关联产出：X篇内容 / X张图文 / X个视频
💡 建议：XXX 可考虑降级/取消
```

### 模块 C · 内容复盘

文章发出后 48 小时，自动拉数据，跟历史均值对比，告诉你这篇是超预期还是扑了。

**工作流：**
1. 读取最近 48h 内发布的新文章数据
2. 与同类型历史均值对比：
   - 阅读：是否在均值 ±1σ 范围内
   - 转发率：转发/阅读 比值
   - 关注转化：新关注/阅读 比值
   - 互动率：(点赞+在看+评论)/阅读
3. 判定：超预期 / 符合预期 / 低于预期 / 扑街
4. 如果超预期：分析可能的成功因素（标题、选题、发布时间、热点关联）
5. 如果低于预期：分析可能原因，记录教训
6. 写入复盘档案，反哺选题情报模块的历史规律

**输出格式：**
```md
📝 内容复盘 | 《文章标题》发布48h

📊 数据快照
  阅读：XXX（同类均值 XXX，+XX%/-XX%）
  转发：XX（转发率 X.X%，同类均值 X.X%）
  关注：+X（转化率 X.X%，同类均值 X.X%）
  互动：XX（互动率 X.X%，同类均值 X.X%）

🎯 判定：超预期 / 符合预期 / 低于预期

🔍 分析
  成功/失败因素：
  1. ...
  2. ...

📖 教训记录
  - 下次同类选题应该 ...
```

### 模块 D · 多平台增长

追踪公众号、小红书、X/即刻、个人网站的增长数据和平台策略。

**工作流：**
1. 读取各平台最新数据（手动输入或脚本抓取）
2. 按周记录：
   - 公众号：粉丝数 / 本周新增 / 文章数
   - 小红书：粉丝数 / 本周新增 / 图文数 / 互动量
   - X/即刻：粉丝数 / 本周发布数
   - 个人网站（hiloki.ai）：PV/UV（从 analytics.hiloki.ai 获取）
3. 计算周增长率
4. 标记异常波动（突然涨粉/掉粉）
5. 输出周度增长简报

**输出格式：**
```md
📈 多平台增长周报 | 第XX周

| 平台 | 粉丝 | 周增 | 发布数 | 增长率 | 趋势 |
|------|------|------|--------|--------|------|
| 公众号 | 5XXX | +XX | X篇 | +X.X% | ↑ |
| 小红书 | XXX | +XX | X张 | +X.X% | ↑ |
| 网站 PV | XXX | +XX | — | +X.X% | ↑ |

⚠️ 异常：小红书周三单日涨粉 XX（可能是 XXX 图文带来）
💡 建议：本周小红书图文表现好，可加大产出频率
```

## 数据源

| 数据源 | 位置 | 获取方式 |
|--------|------|---------|
| 公众号文章数据 | 飞书多维表格（165篇） | 飞书 API 读取 |
| AI 订阅积分 | 飞书「AI 积分追踪」表 | 飞书 API 读取 |
| 小红书数据 | 手动输入 / 界面截图 | 用户提供 |
| 网站 PV/UV | analytics.hiloki.ai/stats | HTTP GET |
| 实时热点 | web_search 搜索 | AI 领域关键词 |
| 历史选题 | OB 📝 选题池/ | 本地文件读取 |

## 使用规则

1. **不编造数据**。数据源缺失时明确标注"数据不足"，不给假数字
2. **统计显著性**。样本量 < 10 的分类标注"统计意义有限"
3. **先问后跑**。首次使用时先确认数据源是否可用，不要假设飞书表格还存在
4. **不过度推荐**。如果数据不支持强推荐，就说"数据不足，建议持续追踪"
5. **复盘不讨好**。低于预期就是低于预期，不粉饰数据
6. **定时触发**。建议 cron：每周一 9:00 跑选题情报 + 增长周报，文章发布后 48h 跑内容复盘
