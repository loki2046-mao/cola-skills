# FreeView 文字形态风格参考

本文件仅在 `artifact_type=text` 时参考。网页形态不适用。

## 核心原则

文字形态的作品是"读后感"的升级版——保留用户的声音，把零散的感受凝聚成有结构的东西。

关键词是"凝聚"——用户口喷是散的，你把它收拢、塑形、给个结构，但不换掉材质。

## 开头不从"今天看了"开始

从感受直接切入。用户口喷时说的第一句话往往就是最好的开头。

## 情绪先于分析

先说感受，再展开。不要先给结论。

## 保留口语感

口喷时的口语词（就、反正、你说、那种、嘛、呢）可以在作品里出现。不要全部洗掉。

## 标点有情绪

感叹号、问号、省略号、波浪线——这些是情绪的载体。只有逗号句号的文字是死文字。

## 标题从感受里长出来

不要格式化标题。用户口喷里如果有特别好的一句话，那句话就是标题。

## 要避免的 AI 味

1. "不是A而是B"句式及变体 → 改成正面表述
2. 破折号过多 → 用逗号、句号、括号替代
3. 短句句号连击 → 改成有连词的长短句交替
4. 每句都有"重量感" → 改成更轻的说法
5. "赋能/抓手/闭环/颗粒度/底层逻辑" → 全删
6. "接住了" → 不用
7. "......的那种"句式 → 太 AI，改成具体描述
8. 标点单调 → 加情绪标点
9. 每段都像在说真理 → 加碎碎念、括号吐槽、口语口癖，允许有"废话感"

## 事实使用

Step 2 核实过的事实自然嵌入正文，不要像百科条目一样列举。不要写"基本信息卡"。

## 结构参考（非硬性）

- 结构 A — 情绪递进：冲击开头 → 展开 → 深入 → 着陆
- 结构 B — 碎片串联：核心意象 → 多个碎片段落 → 暗线连接 → 呼应开头
- 结构 C — 对话体：自问自答 → 在问答中展开 → 不一定有结论 → 结尾是新问题
