# 场景映射 · scene-mapping.md
> 做任何设计之前必读这个文件。
> 不同主题/内容/用途 → 对应不同的底色方案、布局节奏、组件选择、风格语言。
> 千万不要把 Landing Page 的架构套在产品介绍上，也不要把产品介绍套在封面上。

---

## 核心判断问题（做设计前先回答这三个）

1. **这个内容是关于「人」还是关于「产品/工具」？**
   - 关于人（个人主页/个人品牌）→ 有大量留白，文字就是主角，情绪感强
   - 关于产品 → 功能要可见，截图要出现，操作流程要展示

2. **用户来这里的目的是什么？**
   - 了解这个人是谁 → 个人主页型
   - 了解这个工具能做什么 → 产品介绍型
   - 做决定要不要用/关注 → CTA 型，转化为核心

3. **内容的情绪温度在哪里？**
   - 参考 design-dna.md 第 10 节「情绪色谱」
   - 冷（行业观察/分析）→ 偏克制、硬边框、小字号
   - 暖（生活/工具/教程）→ 浅底、圆润感、适中字号

## 场景一：个人主页 / 个人品牌页

**代表案例：** hiloki.ai 个人 Landing Page

### 底色方案
- Hero：深底 `--dark2` + 大量留白
- Statement/Stats：浅底 `--warm` 或 `--paper`
- Works：深底 `--dark`
- About：深底或浅底（与 Works 相反）
- Footer：深底

**规则：深浅必须交替，连续两个深色区 = 节奏死**

### 布局节奏
- Hero：全高视口，左文字右 IP 形象，底部左对齐
- Stats：网格线分割数字大字报，不是卡片
- Works：杂志式左右交替（图左信息右 / 图右信息左），每行不重样
- About：左侧细线数据列 + 右侧大字标题

### 核心组件
| 区块 | 组件 | 为什么 |
|------|------|--------|
| Hero | 极大标题（100px+）+ 极细副文字 | 字号反差是记忆点 |
| Stats | Space Grotesk weight 200 超细大数字 | 数字当图用 |
| Works | 真实产品截图 + 杂志交替布局 | 真实感 > 插图 |
| 标签 | 无底色细边框 pill | 不抢主体 |
| CTA | 无圆角或极小圆角 | 有力量感 |

### 禁止
- 居中对称布局（除 Hero 首屏一次）
- 图标插画替代真实截图
- 每个区块节奏相同

## 场景二：产品介绍页 / 工具介绍页

**代表案例：** NovelFlow 介绍页、公众号排版编辑器介绍页

### 与个人主页的核心差异
- 个人主页：你是主角，留白是主旋律
- 产品介绍：产品功能是主角，截图/操作界面必须出现
- 产品介绍要「让人看完觉得我要用这个」，不是「让人觉得这个人好厉害」

### 底色方案
- Hero：深底，但比个人主页更有「产品感」（可以加界面截图或 mockup 元素）
- 痛点/问题区：浅底，情感共鸣，字要大
- 功能区：深底，功能要可视化，不只是文字列表
- 数据/成就区：浅底大数字
- CTA：最深的深底，强调转化

### 布局节奏（和个人主页不同！）
- Hero：「一句话命中痛点」的标题 + 产品截图或动态展示（不是 IP 形象）
- 问题区：3 列卡片展示痛点，有数字序号，浅底
- 功能区：**左右对分**——左侧功能名+描述，右侧具体 feature tag 列表（不是文字段落）
- 截图展示：大图 + 说明文字，图才是主角
- CTA：全宽暗底，一个大按钮，不要分散注意力

### 核心组件
| 区块 | 组件 | 为什么 |
|------|------|--------|
| Hero | 产品截图/界面 mockup | 让人一眼知道这是什么 |
| 痛点 | 数字序号 + 短标题 + 1-2 句描述 | 快速命中，不废话 |
| 功能 | 左标题/右 tag 列表，交替排布 | tag 比段落更好扫视 |
| 数字 | Space Grotesk 200 大数字 | 建立可信度 |
| CTA | 居中单按钮 + 极小补充说明 | 聚焦转化 |

### 禁止
- 功能区全是文字段落（要有可视化元素）
- 复制个人主页的「杂志交替布局」套在功能上（功能要左右分，不是图文交替）
- Hero 放 IP 形象（Hero 要放产品本身）

## 场景三：演讲 Slides / 分享 PPT

**代表案例：** 公众号话题分享、AI 工具教程

### 底色方案
- 封面页：全深底，大标题
- 章节幕封：深底，中等大标题 + 小序号
- 内容页：浅底（暖米），正文可读
- 数据页：深底，大数字
- 结尾页：全深底，CTA

### 布局节奏
- 一页一件事，不堆多个要点
- 横向翻页，不竖向滚动
- 每页必须有一个「视觉主角」（大字/大图/大数字选一个）
- 过渡页用章节幕封隔断节奏

### 核心组件
| 页型 | 组件 |
|------|------|
| 封面 | 极大得意黑标题 + 极细 eyebrow |
| 内容 | 左侧大标题 + 右侧要点列表（最多5条）|
| 数据 | 超细数字 + 极小说明 |
| 引用 | 左橙竖线 + 大字金句 + 极细署名 |
| 结尾 | 全深底 + 中央CTA按钮 |

**详见 references/cover-templates.md**

### 快速判断
| 文章类型 | 用哪套 |
|---------|--------|
| 观点/思考/评论 | 深底 A（暗底+琥珀橙高亮词） |
| 教程/工具/测评 | 浅底 B（奶杏底+右橙竖条） |
| 开门见山强观点 | 全橙底 C（橙底白字） |

## 场景五：小红书/社媒卡片

### 底色方案
- 暖米 `--warm` 为主，干净不花哨
- 关键数据/金句可用全橙底

### 布局
- 竖版 3:4，顶部标题 + 中部内容 + 底部品牌
- 字号极端反差（标题超大，正文超小）
- 每张卡片只有一个视觉重心

## 不同场景的颜色侧重

| 场景 | 主底色 | 品牌橙用途 | 霓虹青用途 |
|------|--------|-----------|-----------|
| 个人主页 | 深底为主，浅底交替 | 标题关键词、按钮 | Hero IP 光晕（一次） |
| 产品介绍 | 深底为主，痛点区浅底 | 功能标题高亮 | 极少，技术感场景 |
| Slides | 深浅交替，封面深 | 章节序号、强调词 | 数据页点缀 |
| 封面 | 见 cover-templates.md | 主色/竖条/全底 | 不用 |
| 社媒卡片 | 暖米为主 | 数字、标题点缀 | 不用 |

## 检查自己做得对不对

做完一个设计，问自己：

1. **我用的布局/组件是这个场景专属的，还是从另一个场景复制来的？**
   - 把个人主页的「杂志交替」套在产品功能上 = 错
   - 把产品介绍的「功能 tag 列表」套在个人主页上 = 错

2. **这个设计放到对应场景里，用户的核心行为是什么？**
   - 个人主页：看完之后「哦，这是这样一个人」
   - 产品介绍：看完之后「我要去试试这个工具」
   - Slides：看完之后「我记住了这几个核心信息」

3. **如果换一个主题放进这套设计，它还成立吗？**
   - 成立 = 设计系统做对了
   - 不成立 = 你只是做了一个一次性的页面，不是设计系统
