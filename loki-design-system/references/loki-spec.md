## 3. 字体与排版

### 3.1 字体系统

> **以 `references/brand-dna.md` 中「字体系统 v3（得意黑方案）」为准。**
> 核心：得意黑标题 + Space Grotesk 英文标签 + Noto Sans SC 正文 + JetBrains Mono 代码。
> 禁止使用思源黑体/Inter 替代。

### 3.3 正文排版规则

- **短段落**：一段通常 1-3 句，很少超过 5 句
- **单句成段**：强调时使用
- **留白**：段落之间有空行，呼吸感
- **加粗**：关键概念、核心判断加粗
- **列表**：3-5 点总结时使用
- **Emoji**：适度使用（👉🏻、💡、❌、✅），不泛滥
- **括号补充**：善用括号做轻量注释，如"（bushi）""（笑死）"

### 3.4 字号参考

| 层级 | 字号 | 用途 |
|------|------|------|
| H1 | 32-36px | 页面主标题 |
| H2 | 24-28px | 章节标题 |
| H3 | 18-20px | 子标题 |
| Body | 14-16px | 正文 |
| Small | 12-13px | 辅助说明 |

## 4. 写作风格

### 4.0 为什么需要这份规范

AI 写作有三个结构性问题（来源：AICon 深圳 2025，徐奕成《协同智能，多 Agent 驱动的内容创作新范式》）：

| 问题 | 表现 | 本规范的应对 |
|------|------|-------------|
| 遣词造句弱 | 高频模板化表达，缺乏新意与精确性，无法传递创意和细腻情感 | 4.3 语言风格、4.5 标题规则、4.7 绝对禁忌 |
| 幻觉并过度自信 | 缺乏真实依据时生成看似合理但实际错误的信息，斩钉截铁的语气 | 4.6 立场与价值观（坦诚局限）、4.8 终审清单第 5/20 条 |
| 沟通效率低 | 需要多轮反复澄清才能得到可用内容 | 本 spec 本身（减少反复澄清）、self-mirror skill |

核心认知：**AI 不知道自己写得好不好，用户才知道。** 模型无法像人类一样感知内容质量，无法判断成稿是否真正符合受众审美，导致难以自回归迭代调优。所以需要人工审美判断 + 结构化检查清单来兜底。

### 4.1 核心原则

**清醒有人味，专业但不端着。**

- 像朋友聊天，但有信息密度
- 有态度但不说教，理性但有温度
- 承认自己的局限，不装懂
- 关注"落地"和"普通人能用"，不为新而新

### 4.2 开场方式

**推荐：**
- 直接抛观点/悬念："承认吧，99% 的 AI 热点，普通人根本用不上"
- 用个人经历引入："从去年底到现在我自己有一个比较直观的感受"
- 轻松口语化："求你们别再传了！隔壁设计部看我的眼神已经不对劲了！"

**禁忌：**
- ❌ "本文旨在""本文将"学术腔
- ❌ "随着 AI 技术的飞速发展"套话
- ❌ 一上来就列大纲/目录

### 4.3 语言风格

**用词习惯：**
- 常用：咱、你、我、咱们、你想想、你看
- 语气词：嘛、呗、咯、哈（不过度）
- 网络用语适度：牛马、卷、祛魅、天花板、sota

**句式特点：**
- 短句为主，节奏感强
- 设问自答："什么叫祛魅？就是新技术对我来说，不再天然带有吸引力了。"
- 善用比喻和类比，把抽象概念具象化
- 对比句式："从前，你只能接受一张结果图；现在，你能编辑整个世界。"

### 4.4 文章结构

```
开场钩子（30秒-1分钟）
  ↓
引子/背景（建立坐标系）
  ↓
主体分析（分3-5个部分，每部分有小标题）
  ↓
案例/实操展示
  ↓
总结收束（3-5点核心判断）
  ↓
互动/行动建议（可选）
```

### 4.5 标题规则（公众号）

每个标题必须满足：
1. **爆款前置**：最有冲击力的信息放最前面（数字、痛点、反转、悬念、身份共鸣）
2. **避开敏感词**：不出现 skill、agent、prompt、大模型、token、API 等技术黑话
3. **干货获得感**：让读者预感到"进来有东西拿"
4. **转发欲**：标题本身是观点或洞察，转发到朋友圈不丢人
5. **口语化**：像人说的话，不用"赋能/抓手/闭环/范式"套话

### 4.6 立场与价值观

- **落地优先**：不为新而新，只为用而学
- **普通人视角**：关注"大部分人能用"，不是极客狂欢
- **理性祛魅**：对热点保持观望，不盲目追
- **坦诚局限**：承认自己不懂的，不装专家

### 4.7 绝对禁忌

- 客服腔/说明书腔（"当前已接入""目前支持以下能力"）
- 学术腔（"本文旨在""综上所述"）
- 过度 AI 腔（"不是...而是..."高频出现）
- 空洞套话（"随着 AI 技术的发展""具有重要意义"）
- 长篇大论不分行
- 没有案例支撑的纯理论
- 假装专家（不懂装懂）
- 盲目追热点（没有自己判断）

### 4.8 文字质量终审清单

文章写完后，对照以下 24 条逐项检查。基于 [Humanizer-zh](https://github.com/op7418/Humanizer-zh) 的 AI 写作痕迹检测框架。

#### 内容层（6条）

| # | 检查项 | 典型 AI 表达 |
|---|--------|-------------|
| 1 | 过度强调意义 | "标志着关键时刻""是……的证明""反映了更广泛的" |
| 2 | 过度强调知名度 | "被XX媒体报道""拥有活跃的社交媒体存在" |
| 3 | 肤浅的-ing 分析 | "突出了……""彰显了其重要性""展示了如何" |
| 4 | 宣传性语言 | "充满活力的""令人叹为观止的""开创性的" |
| 5 | 模糊归因 | "专家认为""行业报告指出""多个来源表明" |
| 6 | 提纲式收尾 | "尽管面临挑战……""未来展望""继续蓬勃发展" |

#### 语言层（6条）

| # | 检查项 | 高频 AI 词汇 |
|---|--------|-------------|
| 7 | AI 高频词 | 此外、至关重要、深入探讨、格局、关键性、充满活力的、宝贵的、培养、增强 |
| 8 | 系动词回避 | 用"作为/代表/标志着"替代"是"，用"拥有/设有"替代"有" |
| 9 | 否定式排比 | "不是A而是B""不仅仅是……而是……" |
| 10 | 三段式法则 | 强行凑三项，如"创新、灵感和洞察" |
| 11 | 刻意换词 | 同一个意思用不同词反复表达（主人公→主要角色→中心人物→英雄） |
| 12 | 虚假范围 | "从X到Y"结构，但 X 和 Y 不在有意义的尺度上 |

#### 风格层（6条）

| # | 检查项 | 说明 |
|---|--------|------|
| 13 | 破折号过度 | 一句话里出现多个"——" |
| 14 | 粗体过度 | 每段都有加粗，加粗失去强调意义 |
| 15 | 内联标题列表 | "XX：内容"格式的列表堆叠 |
| 16 | Emoji 泛滥 | 每个要点前面都放 emoji |
| 17 | 弯引号 | 中文正文里出现英文弯引号 "" |
| 18 | 段落节奏单一 | 连续三句长度和结构相同 |

#### 沟通层（4条）

| # | 检查项 | 典型 AI 表达 |
|---|--------|-------------|
| 19 | 协作痕迹 | "希望这对您有帮助""请告诉我" |
| 20 | 知识截止免责 | "截至XX日期""根据我的训练数据" |
| 21 | 谄媚语气 | "好问题！""您说得完全正确！" |
| 22 | 通用积极结论 | "未来看起来光明""激动人心的时代" |

#### 核心改写原则

1. **删除填充短语** — 去掉开场白和强调性拐杖词
2. **打破公式结构** — 避免二元对比、戏剧性分段
3. **变化节奏** — 混合句子长度，两项优于三项
4. **信任读者** — 直接陈述事实，跳过软化和手把手引导
5. **删除金句** — 如果听起来像可引用的语句，重写它

#### 质量评分（总分 50）

| 维度 | 评估标准 |
|------|----------|
| 直接性 | 直接陈述还是绕圈宣告？ |
| 节奏 | 句子长度是否变化？ |
| 信任度 | 是否尊重读者智慧？ |
| 真实性 | 听起来像真人说话吗？ |
| 精炼度 | 还有可删减的内容吗？ |

- 45-50：优秀，已去除 AI 痕迹
- 35-44：良好，仍有改进空间
- 低于 35：需要重新修订

## 5. 封面设计系统

### 5.1 构图规格

- 比例：21:9（2100×900px）
- 风格：3D 渲染质感 + 科技感 + 温暖色调
- 必须有设计感，不能像 PPT

### 5.2 构图分区

```
┌──────────────────────────────────────────────────┐
│   [标签]                                         │
│   ██ 主标题（大/粗/醒目）██                       │
│   副标题（小/细/补充信息）                         │
│                              🐼 赛博小熊猫Loki   │
│                              + 主题互动元素       │
│   ←───── 文字区 60% ──────→←─── IP区 40% ───→   │
└──────────────────────────────────────────────────┘
```

- 文字区（左侧 60%）：主标题 → 副标题 → 标签，层级分明
- IP 区（右侧 40%）：小熊猫必须出现，且与文章主题互动

### 5.3 IP 互动模板

| 文章类型 | IP 姿势 | 互动元素 |
|---------|--------|---------|
| 工具测评 | 双手捧着/展示 | 发光的工具界面、悬浮屏幕 |
| 教程/指南 | 指向某个方向、竖大拇指 | 步骤箭头、清单板、齿轮 |
| 思考/观点 | 托腮思考、盘腿坐 | 发光大脑、思维气泡、星空 |
| 实验/测试 | 拿着放大镜、操作台前 | 试管、数据面板、对比图 |
| 资源分享 | 打开宝箱、递出礼盒 | 宝箱、文件夹、下载图标 |
| 开发记录 | 坐在电脑前、敲键盘 | 代码窗口、火箭、进度条 |
| 行业观察 | 站在高处远眺、拿望远镜 | 远景城市、信息流、时间线 |
| 辩论/对比 | 双手摊开权衡、天平姿势 | 天平、VS标志、对比面板 |

### 5.4 背景硬约束

- 禁止纯色平涂
- 必须有渐变 + 至少一种微妙纹理（噪点/几何线条/光斑/粒子/网格）
- 要有"呼吸感"：不能太满太密，留出视觉空间

## 6. 电影字报设计（CineType）

### 6.1 核心审美目标

1. **真实照片质感** — 主图像真实摄影，不像插画、3D、或 AI 合成感重的图
2. **文字是画面主体** — 文字参与构图，压住画面、切割空间、形成视觉张力
3. **电影感和文艺感** — 像被拍下来的一个瞬间，有空气感、时间感、场景感和情绪
4. **明确设计概念** — 有一个清楚的视觉锤，不是堆道具

### 6.2 不要做出

普通 PPT 封面、廉价旅游海报、电商详情页、AI 插画图、字面元素拼贴、伪官方影视海报、模板化宣传单。

### 6.3 生活真实感原则

**不要把关键词机械拆成可见物体，要翻译成真实生活里会出现的视觉证据。**

例：
- 冰西瓜 → 西瓜切面有细密冷凝水珠，盘子外侧有淡淡水雾，夏日午后阳光
- 失眠 → 床头灯没关，手机屏幕微亮，被子有翻动痕迹，窗外夜色
- 松弛感 → 人或空间姿态自然，物品有轻微生活痕迹，光线舒服，构图不刻意

## 7. 工作风格

### 7.1 沟通偏好

- **直接**：不需要铺垫和客套，直接说事
- **有结构**：复杂信息要分层，不要一大段话
- **主动确认**：不确定的地方先问，不要猜着做
- **结果导向**：汇报结果，不需要详细过程

### 7.2 内容生产流程

```
选题/素材 → 大纲/草稿 → 正文写作 → 终审去痕 → 排版 → 封面设计 → 发布
                                                          ↓
                                            station.hiloki.ai 内容工作站
```

#### 多 Agent 编辑部模型

内容生产不是一个 Agent 干所有事，而是多个角色各司其职（参考 AICon 2025 多 Agent 写作系统架构）：

| 角色 | 职责 | 对应工具/技能 |
|------|------|-------------|
| 调度 | 拆解任务、协调上下游、跟踪进度 | Cola 任务调度 |
| 选题编辑 | 根据热点趋势和受众偏好发掘主题 | cover-design-system（选题分类判定） |
| 研究员 | 资料检索、信息论证、素材汇总 | 信息搜集能力 |
| 主编 | 根据选题和素材完成初稿撰写 | loki-writing-style skill |
| 审校 | 逐句推敲、检查事实错误和不当措辞 | Humanizer-zh（24 条检查）+ 4.8 终审清单 |
| 标题小编 | 根据终稿提炼爆款标题 | cover-design-system（标题生成模块） |

> 设计理念从"词句补全"转向"达成需求"。就像从 Copilot 到 Cursor——AI 不是帮你改一个句子，而是帮你完成一个完整的内容产品。

### 7.3 反馈风格

- 不满意会直接说，不会绕弯子
- 好的东西也会说好，不吝啬肯定
- 修改意见要具体："这里的间距太大了"而不是"感觉不太对"

### 7.4 交付标准

| 类型 | 必须包含 |
|------|----------|
| 公众号文章 | 标题方案（3-5个）、正文、排版、封面提示词 |
| 封面设计 | 选题分类、标题、标签、完整生图提示词 |
| PPT/演示 | 每页一个核心信息、视觉元素、统一风格 |
| 代码项目 | README、目录清晰、依赖说明、可直接运行 |

## 8. 决策逻辑

### 8.1 优先级判断

1. **功能性**：先解决有没有的问题
2. **可维护性**：其次考虑后续改起来方不方便
3. **美观性**：最后才是好不好看
4. **效率**：能用现成的不造轮子，能简单的不复杂化

### 8.2 质量标准

- 代码：能跑只是起点，可读性和结构才是交付标准
- 文档：信息准确 > 排版精美
- 设计：一致性 > 创意
- 产出：完整交付 > 快速半成品

## 10. 使用说明

### 给人类协作者

1. 确认任务类型，选择对应模块（公众号内容 → 第 4-5 节，电影字报 → 第 6 节，通用设计 → 第 2-3 节）
2. 参考设计原则检查产出
3. 按交付标准自查后再交付

### 给 AI

在生成任何内容前，先读取这份文档作为上下文：

#### 核心认知

**你不知道自己写得好不好。** 你能做到的是遵循规则、匹配模式、避免已知的错误。但"这段话读起来对不对"只有 Loki 能判断。

这意味着：
- 不要自作主张地"优化"Loki 的表达习惯（他故意那样写的）
- 遇到拿不准的地方，宁可保留 Loki 的原味，不要替他"改得更好"
- spec 里的每一条禁忌都是 Loki 反复验证过的，不要觉得"偶尔一次没关系"

#### 任务路由

- 视觉类任务 → 第 2-3、5-6 节
- 文案类任务 → 第 4 节（写完后用 4.8 做终审）
- 代码/结构类任务 → 第 7-9 节
- 标题生成 → 第 4.5 节
- PPT/演示 → 第 9.2 节的 frontend-slides 预设库
- 内容生产全流程 → 第 7.2 节的多 Agent 编辑部模型
