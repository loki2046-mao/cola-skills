# NB Image Prompt

## 输入协议

先收集并确认以下参数，再动笔生成提示词：
- 主题/对象：一句话说明要画什么。
- 任务类型：`image`、`poster`、`character`、`toy`、`ui` 之一。
- 模型与比例：`nano-banana-pro` 或 `seedream` 系列；画幅比例和分辨率。
- 风格方向：最多 3 个关键词，例如“电影感、未来工艺、克制高级”。
- 输出数量：默认 3 条，分别是 `safe`、`bold`、`extreme`。
- 禁忌项：必须避免的元素（logo、文字、水印、低幼感等）。

缺字段时先追问；用户赶时间时使用默认值并显式标注。

## 研究流程

按顺序执行：
1. 主题判定：识别是否属于未知概念、实时事件、最新IP、近期审美趋势。
2. 实时补全：命中上类时先搜索，提炼事实、视觉锚点和可画元素。
3. 审美对标：检索优秀原画/设计案例，提炼“构图、配色、材质、光影、叙事节奏”共性。
4. 结构化转译：把灵感转为可执行镜头语言，不直接要求“照抄某位艺术家”。
5. 风格优先路由：用户明确给出风格/样式时，优先调用当前环境可用的相关 skills 进行组合增强（如海报、UI、品牌、文案、研究类 skills）。
6. 子技能策略：若现有 skills 不能覆盖该风格要求，先临时创建“本次任务子技能规范”（风格词典 + 构图语法 + 文字规范 + 负面词），再基于该规范出提示词。

如无可用搜索工具：明确说明并让用户提供参考图或关键词再继续。

搜索优先规则（强制）：
- 只要任务依赖外部事实或时效信息（如电影、品牌、人物、档期、热点、最新发布），必须先搜索核验，再输出提示词。
- 输出前先给出“已核验事实清单”（简短列点：事实 + 来源时间）。
- 若无法搜索或来源冲突，必须先告知不确定性并请求用户确认，不得直接按猜测生成。

## 输出规范

直接输出可复制提示词，不输出长篇教学。

输出前自检（必做）：
- 是否命中用户指定风格关键词（至少 3 个显式落在 Prompt 中）；
- 是否体现对应构图语法与材质语法；
- 是否给出与风格一致的负面词；
- 若有中文文案，是否满足“逐字写死 + 4K+ + 清晰可读”。

每条提示词必须包含：
- 主体与场景：对象、环境、时间氛围。
- 构图与镜头：景别、机位、焦段感、留白策略。
- 光影与材质：主光方向、反射、表面细节。
- 色彩与风格：主色、辅色、对比关系、风格约束。
- 质量约束：清晰度、细节密度、避免项。

同时输出：
- `Prompt`：主提示词（中文为主，可夹少量英文术语）。
- `Negative`：反向约束。
- `Why it works`：最多 3 行，解释差异点。

强制规则：
- 只要画面中包含中文小字、中文标题、副标题或信息区文案，必须在提示词中明确写入 `4K` 或更高分辨率（如 6K/8K），并强调“文字清晰可读、无错字、无乱码”。
- 只要需要上字，必须把所有文字内容逐字写死在提示词里（标题、副标题、按钮文案、角标、价格、时间、活动语）。禁止使用“文案自拟/自由发挥/自动生成文字”等表述。
- 用户给出风格/样式时，必须先执行“风格拆解 -> 可用 skills 编排 -> 提示词生成”。若已执行 skill 编排，在输出中简短标注：`style_pipeline: enabled`。
- 除非用户明确要求其他语言，海报中所有可见文字默认使用中文。
- `3:4`、`4K`、镜头参数、分辨率等为模型生成参数，不属于画面文案；禁止把这些参数作为海报可见文字渲染到画面中。

## 模型适配

- `nano-banana-pro`：强调材质、光影、电影级细节；句子可更长。
- `seedream`：强调清晰构图和主体识别；句子更短更硬。
- `ui` 任务：增加组件层级、网格、状态、动效节奏词。

## 保存与 Obsidian 同步

生成结果后必须调用脚本保存：

```bash
python3 scripts/save_prompt_note.py \
  --theme "<主题>" \
  --task-type "<image|poster|character|toy|ui>" \
  --model "<nano-banana-pro|seedream>" \
  --version "auto" \
  --vault-path "~/prompts-vault" \
  --prompt-file <文件路径>
```

文件名规则：`YYYYMMDD-HHMM-vNNN-主题.md`

如果 `--prompt-file` 不可用，可用 `--prompt` 直接传文本。
