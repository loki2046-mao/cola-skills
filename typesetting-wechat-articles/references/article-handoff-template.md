# 公众号文章交付模板

这个模板是给“排版前交稿”用的。

目标不是写得很复杂，而是让我准确知道：

- 哪些是标题
- 哪些是正文
- 哪张图跟哪一段走
- 哪些内容要做特殊模块

## 最小可用版本

```md
# 文章标题

> 风格偏好：白底、局部橙黄色、偏杂志感
> 输出模式：原生稳定 / SVG-first / SVG 交互画廊

## 第一节标题

这里是正文第一段。

这里是正文第二段。

[配图]
- img-01-cover.jpg

## 第二节标题

这里是正文。

[组图]
- img-02.jpg
- img-03.jpg
- img-04.jpg
```

## 完整推荐版本

```md
# 文章标题

> 风格偏好：白底、局部橙黄色、偏杂志感
> 输出模式：原生稳定
> 是否接受整篇 SVG：接受 / 不接受
> 是否接受滑动画廊：接受 / 不接受

## 封面说明

[配图]
- img-01-cover.jpg

## 第一节标题

这里是正文第一段。

这里是正文第二段。

[重点词]
- 这个词
- 那个词

[配图]
- img-02-chat.png

## 第二节标题

这里是正文。

[组图]
- img-03-case-a.jpg
- img-04-case-b.jpg
- img-05-case-c.jpg

[说明]
- 这 3 张图属于同一组内容
- 如果横竖比例不一致，请按横版组和竖版组拆开

## 第三节标题

这里是正文。

[序列卡片]
1. 第一步：先做什么
2. 第二步：再做什么
3. 第三步：最后做什么

## 第四节标题

这里是正文。

[长文本卡片]
这里放长提示词、长摘录、长模板、长方法论。
如果内容很长，默认做成有限高度的卡片，
内部允许纵向滚动；
如果是超长单行内容，允许横向滚动。
[/长文本卡片]
```

## 图片文件夹建议

目录结构建议：

```text
你的文章文件夹/
├── article.md
└── images/
    ├── img-01-cover.jpg
    ├── img-02-chat.png
    ├── img-03-case-a.jpg
    ├── img-04-case-b.jpg
    ├── img-05-case-c.jpg
    └── optional/
        ├── unused-01.jpg
        └── unused-02.jpg
```

## 交稿时最重要的 5 件事

1. 标题要用明确层级，不要把强调句混进标题里
2. 图片必须用稳定文件名，不要只说“这张图”“上一张图”
3. 多图场景要说明是否同组
4. 长文本、提示词、代码块要单独标注
5. “第一点/第二点/第三点”这类序列内容，最好单独标成 `[序列卡片]`
