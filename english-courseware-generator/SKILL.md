# English Courseware Generator

**核心定位**：输入主题 + 选择组件 → 生成生图提示词 → 直接出课件图。

目标效果：适合 6-12 岁儿童的英语学习课件，双语教学，卡通可爱风格，一张图就是一页课件。
不要做出：纯文字 PPT、廉价模板感、没有视觉层次的练习卷。

---

## 一、工作流程

### Step 1: 收集信息

向用户确认以下参数，缺失项使用默认值：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 主题类型 | IP 主题 或 原创主题 | 由用户指定 |
| 主题名称 | IP名称（如 Zootopia）或原创主题名（如 Fruit Kingdom） | 必填 |
| 角色 | 主题中的角色列表（1-6个） | 必填 |
| 学习内容 | 本课的具体知识点（如 Animals, Present Continuous, Fruits） | 必填 |
| 目标年龄 | 6-8 / 8-10 / 10-12 | 6-12 |
| 难度 | Beginner / Intermediate / Advanced | Beginner |
| 画幅比例 | 3:4（竖版课件推荐）/ 16:9 / 1:1 / 4:3 | 3:4 |
| 风格 | cute_cartoon / flat_design / hand_drawn | cute_cartoon |
| 组件选择 | 用户指定组件 或 由skill根据学习目标推荐 | 推荐组合 |

用户赶时间时用默认值并标注。不确定主题角色时先追问。

### Step 2: 选择组件

根据学习目标推荐组件组合。用户也可自行指定。读取 [components.md](./references/components.md) 获取全部27个组件的详细说明。

**推荐组合逻辑：**

- **词汇入门课**：Header + Vocab Cards + Picture-Word Match + Fill Blanks + Footer
- **对话练习课**：Header + Comic Dialogue + Role Play + Daily Expressions + Footer
- **语法句型课**：Header + Grammar Points + Sentence Patterns + Comic Dialogue + Fill Blanks + Footer
- **综合复习课**：Header + Vocab Cards + Multiple Choice + Matching + Word Search + Mini Quiz + Footer
- **趣味游戏课**：Header + Word Bingo + Word Search + Maze Game + Footer
- **阅读写作课**：Header + Reading Comprehension + Writing + Self Check + Footer

用户可自由混搭，但一张课件图建议放 3-6 个组件（太多会拥挤）。

### Step 3: 生成课件内容

根据选定的组件，生成实际的英语教学内容。

读取 [quality-assurance.md](./references/quality-assurance.md) 获取完整的内容质量保证体系——难度校准表、内容自检清单、组件间内容关联规则、文字量适配规则、失败诊断策略、模型适配策略、网格生成算法、多页学习路径规划。

**两种内容模式：**

**模式A：AI自动生成（默认）**
用户只给主题和知识点，AI帮你生成所有内容。适合不知道教什么、或者想快速出课件的场景。

1. **确定词汇集**：根据主题和知识点生成4-8个核心词汇，每个配中文翻译。词汇难度参照难度校准表匹配目标年龄。
2. **对话**：围绕主题角色和知识点编写简短对话（每格1-2句，全页≤8句）
3. **练习题**：根据词汇和语法点生成题目，确保答案唯一、难度递进、答案分布随机
4. **语法**：如涉及语法点，用一句话讲清规则，例句用主题角色和本课词汇
5. **游戏**：网格类组件（Word Search/Bingo）需先用算法生成正确内容，再逐字写入提示词

**模式B：用户提供内容**
用户有明确的题目、词汇、对话内容，直接喂给Skill。适合老师有自己的教学计划、或者家长有明确练习目标的场景。

1. 用户提供的词汇、题目、对话等内容直接使用，跳过内容生成
2. Skill仍然负责：组件排版、ZONE布局、提示词构建、质量检查
3. 如果用户提供的内容有明显的英语拼写错误或教学逻辑问题，提醒用户但不擅自修改
4. 用户提供的内容不足时（比如选了4个组件但只给了2个组件的内容），询问用户补充或由AI补全

**组件间内容关联**：无论哪种模式，同一张课件图上的所有组件共享同一个词汇集。内容按教学逻辑流动：展示→识别→应用→检测→输出。详见 quality-assurance.md 的组件间内容关联规则表。

**内容自检**：生成完内容后（或收到用户提供的内容后），逐项检查 quality-assurance.md 的内容自检清单——英语准确性、教学逻辑性、数量合理性。自检不通过的修正后再进入Step 4。

### Step 4: 构建生图提示词

读取 [prompt-templates.md](./references/prompt-templates.md) 获取提示词模板体系。

核心原则：
1. **所有文字逐字写死** — 标题、单词、题目、选项、对话内容全部写在提示词中，禁止"文案自拟"
2. **文字清晰可读** — 强调 4K 分辨率、文字清晰无乱码
3. **布局分区明确** — 用 ZONE 系统描述每个组件的位置和区域
4. **风格统一** — 色彩、字体风格、边框样式全图统一

提示词结构：
```
[画面整体描述 + 风格]
[排版布局 ZONE 系统 — 每个组件对应一个 ZONE]
[具体文字内容 — 逐字写入]
[色彩与风格约束]
[反向提示词]
[画幅比例]
```

### Step 5: 质量检查

生成提示词后、出图前，逐项确认：

**内容质量**（参照 quality-assurance.md 自检清单）：
- [ ] 所有英文拼写和语法正确？
- [ ] 选择题答案唯一、干扰项合理、答案分布随机？
- [ ] 中文翻译准确无歧义？
- [ ] 词汇难度匹配目标年龄？（查难度校准表）
- [ ] 组件间词汇集共享、内容流向正确（展示→识别→应用→检测）？
- [ ] 网格类组件（Word Search/Bingo）内容用算法生成、逐字写入？

**提示词质量**：
- [ ] 所有文字逐字写死在提示词中？
- [ ] ZONE 布局合理、组件间有明确分隔？
- [ ] 总文字量在安全范围内？（查文字量风险表，>100词需拆页）
- [ ] 风格统一（色彩、字体、边框一致）？
- [ ] 分辨率指定为4K？
- [ ] 反向提示词已附加？

**安全检查**：
- [ ] 适合目标年龄段（无暴力、无不当内容）？
- [ ] IP角色处理正确（动画IP用角色名+外观，真人IP用概念化表达）？

### Step 6: 出图

如果用户要求直接出图，调用图像生成工具。推荐参数：
- 模型：根据用户环境选择。不同模型对文字和布局的擅长程度不同，参照 quality-assurance.md 的模型适配策略
- 比例：3:4（竖版课件标准）或用户指定
- 尺寸：4K（文字更清晰）

出图后检查文字是否清晰可读。如果出问题，参照 quality-assurance.md 的失败诊断与修复策略定位问题并修复：
- 文字乱码 → 检查文字是否逐字写死、文字量是否超限、中英文是否分行
- 文字溢出 → 减少组件数或题量、明确ZONE高度比例
- 布局混乱 → 用ZONE系统描述每个组件位置、添加分隔描述
- 角色不像 → 用角色名+详细外观描述、web_search确认不熟悉的IP
- 风格不统一 → 选定风格预设、统一色彩和边框

多页课件：参照 quality-assurance.md 的多页学习路径规划，确保词汇分布、难度递进、角色和风格统一。

## 二、主题安全规则

### IP 主题

动画IP和真人IP处理方式不同：

**动画IP**（玩具总动员、疯狂动物城、小猪佩奇等）：
- 动画角色没有肖像权风险，可以直接使用角色名和完整外观描述
- 提示词中直接写角色名（如 Woody、Buzz Lightyear、Judy），并配完整外观描述
- 角色外观描述要精准到颜色、服装细节、标志性特征，确保辨识度
- 不确定角色外观时，先 web_search 确认

**真人影视IP**（含真实演员的）：
- 不生成"像某个演员但不准确"的脸
- 用场景、道具、服装局部、背影、空间气质表达
- 目标是概念视觉海报，不是伪造剧照

常见动画IP角色特征参考（不确定时先搜索确认）：
  - Toy Story: Woody（棕帽黄衫红马甲蓝牛仔裤拉绳玩偶）/ Buzz Lightyear（白绿太空服紫头罩紫绿翅膀）/ Jessie（红编辫白底黄花衬衫蓝牛仔裤红帽）/ Rex（绿色霸王龙玩具小手大头）/ Hamm（粉色存钱罐小猪圆身）/ Slinky Dog（弹簧狗）
  - Zootopia: Judy（灰兔子警察制服）/ Nick（红狐狸绿衣黑领带）/ Flash（树懒）/ Clawhauser（胖猎豹）
  - Peppa Pig: Peppa（粉红小猪红裙）/ George（蓝色小恐龙）/ Mummy Pig / Daddy Pig
  - 其他 IP 不确定角色外观时，先 web_search 确认

### 原创主题
- 角色设计要简洁明了，一句话能描述清楚
- 例："Apple Prince — a cute cartoon prince with a red apple-shaped hat"

### 内容安全
- 不包含暴力、恐怖、不当内容
- 角色互动友好、积极
- 适合目标年龄段

## 三、排版系统

读取 [layout-and-style.md](./references/layout-and-style.md) 获取完整排版和风格说明。

### ZONE 系统
每张课件图用 ZONE 系统划分区域：

```
ZONE 1 — 顶部标题区：课程标题（中英文）、主题装饰
ZONE 2 — 主内容区：根据组件数量等分或按比例划分
ZONE 3 — 底部区：页脚、slogan、页码
```

当组件较多时，ZONE 2 内部再细分 sub-zone。

### 风格预设

| 风格 | 色彩 | 边框 | 适用场景 |
|------|------|------|----------|
| cute_cartoon | 粉蓝绿马卡龙色 | 圆角 | 默认，适合低龄 |
| flat_design | 高对比纯色 | 直角 | 现代简洁 |
| hand_drawn | 暖色手绘 | 手绘线条 | 温馨文艺 |

### 色彩系统

- **Pastel（马卡龙）**：适合 cute_cartoon，低龄课程
- **Vibrant（明快）**：适合 flat_design，游戏化课程
- **Warm（暖色）**：适合 hand_drawn，故事类课程
- **Cool Blue（冷蓝）**：适合语法/逻辑类课程

## 四、交互策略

### 用户说"帮我做一个XX主题的英语课件"
1. 确认主题、角色、学习内容
2. 推荐组件组合
3. 生成内容并展示给用户确认
4. 确认后生成提示词
5. 如果用户要求出图，直接调用生图

### 用户说"我只要词汇卡"
直接使用对应单个组件，配 Header + Footer 即可。

### 用户说"组件太多了/太少了"
调整组件数量。一张3:4课件图建议3-6个组件区域。

### 用户说"文字不清楚/乱码"
参照 quality-assurance.md 的失败诊断策略：
1. 检查提示词中文字是否逐字写死
2. 检查单ZONE文字量是否超过安全上限
3. 强调分辨率提升到4K
4. 中英文分行/分区域显示，不要混排同一行
5. 减少单个ZONE内的文字量
6. 如果仍然失败，更换生图模型（参照模型适配策略）
7. 网格类组件（Word Search）确保字母网格逐行写入提示词

### 用户说"风格不喜欢"
切换风格预设或让用户描述期望的风格关键词。

### 用户已有词汇表/内容
直接使用用户提供的内容，跳过 Step 3 的内容生成（模式B），直接进入提示词构建。用户只需说【这是我的词汇/题目/对话内容，帮我排版成课件】即可。

### 用户不确定教什么
直接说主题和知识点即可（模式A），AI自动生成全部内容。比如【帮我做个小猪佩奇的颜色课件，6-8岁】就够了。

### 用户已有部分内容但需要补充
混合模式：用户提供的部分直接用，缺失的部分AI补全。补充前告知用户哪些是AI生成的。

## 五、Skill 设计哲学

这个 Skill 是一个**课件设计与提示词生成系统**，不只是模板填充器。

> 组件拆得细，才能自由组合应对不同题型和训练目标
> 所有文字必须逐字写死在提示词中，否则生图模型会乱码
> 一张课件图就是一页教材，不是一张插画
> 主题不限，组件不限，但布局和风格必须统一
> 课件首先要教学有效，其次才好看
> 开源给别人用，所以流程要清晰、组件要独立、说明要充分

### 下限保证

这个Skill 的下限是70分——不管用哪个生图模型、不管什么主题、不管什么组件组合，产出的课件至少达到以下标准：

1. **教学有效性**：词汇难度匹配年龄、答案唯一正确、组件间内容关联共享、难度递进
2. **文字可读性**：所有文字逐字写死、4K分辨率、文字量在安全范围内、中英文分行
3. **布局合理性**：ZONE系统分区、组件不重叠、3:4竖版3-6个组件、文字量适配
4. **失败修复能力**：有系统性的失败诊断树，能定位问题并给出修复方案
5. **模型适配能力**：根据不同生图模型调整策略，不依赖单一模型

上限取决于用户选用的生图模型和大语言模型的质量。模型越好，文字越清晰、角色越像、布局越精致。但下限由Skill 的规则和流程保证。
