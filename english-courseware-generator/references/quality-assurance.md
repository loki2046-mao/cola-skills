## 内容生成规则

### 词汇生成

生成词汇时遵循以下规则，确保教学有效性：

1. **主题相关性**：每个单词必须与课程主题直接相关，不能凑数。例如主题是"疯狂动物城动物词汇"，词汇应为 Rabbit/Fox/Sloth/Cheetah，不能出现无关的 Cat/Dog。

2. **难度匹配**：词汇难度必须匹配目标年龄和等级，参照难度校准表。

3. **数量合理**：单页课件的核心词汇量建议4-8个。太少信息量不够，太多记不住。

4. **词性搭配**：一节课的词汇尽量包含名词为主，搭配少量动词/形容词，形成可造句的最小集合。例如：Rabbit(名词) + Fast(形容词) + Run(动词) → 可以造句 "The rabbit runs fast."

5. **翻译准确**：中文翻译必须准确，不使用生僻释义。例如 "Right" 在课件中翻译为"正确的"而非"右边"，除非课程主题是方向。

### 对话生成

1. **角色一致**：对话必须使用用户指定的主题角色，不能自创角色。
2. **句型匹配**：对话句型应与课程知识点关联。如果课程教现在进行时，对话里应该用 "I am jumping!" 而非 "I jump!"
3. **长度控制**：每格漫画1-2句对话，全页对话总量不超过8句。长对话拆成多页。
4. **语言自然**：对话要像真实对话，不能是教科书式僵硬句型。
5. **中文翻译**：每句英文对话配一句中文翻译，但翻译放在括号小字或气泡下方，不抢主视觉。

### 练习题生成

1. **答案唯一性**：每道题必须有且只有一个正确答案，不能有歧义。选择题的干扰项必须是合理错误，不能荒谬。
2. **词汇来源**：练习题中出现的词汇应来自本课已教内容，不能出现未教过的新词作为考点。如果是阅读理解，可以出现1-2个新词但需标注。
3. **难度递进**：同一页内的题目从易到难排列。第1题最简单，最后1题最难。
4. **答案分布**：选择题正确答案不能全是同一个选项（如全是B）。随机分布A/B/C。
5. **题干清晰**：题干表述不能有歧义，学生读一遍就知道要选什么。

### 语法讲解生成

1. **讲解简明**：一条语法规则用一句话讲清楚，不超过20个字。例如 "am/is/are + verb-ing = 正在做什么"。
2. **例句匹配**：例句必须用主题角色和本课词汇造句，不能用通用例句。
3. **颜色标注**：语法结构关键部分用颜色高亮，帮助学生识别结构。主语蓝色、谓语红色、宾语绿色。
4. **例外情况**：如果语法有常见例外，简短提及。如 "I am" 不缩写为 "I'm"（在正式写作中）。

## 难度校准表

### 词汇难度分级

| 等级 | 词汇范围 | 示例 | CEFR对应 |
|------|----------|------|-----------|
| Beginner-1 (6-7岁) | 3字母常见词，动物/颜色/数字/家庭 | cat, dog, red, one, mom | Pre-A1 |
| Beginner-2 (7-8岁) | 4-5字母日常词，食物/身体/动作 | apple, jump, happy, hand | A1 |
| Beginner-3 (8-9岁) | 5-6字母，学校/天气/爱好 | school, pencil, sunny, dance | A1 |
| Intermediate-1 (9-10岁) | 6-8字母，抽象词/情感/比较 | beautiful, important, faster, interesting | A2 |
| Intermediate-2 (10-12岁) | 8字母+，学术词/社会话题 | environment, communication, experience | B1 |

### 句型难度分级

| 等级 | 句型范围 | 示例 |
|------|----------|------|
| Beginner-1 | I am ___. This is ___. | I am happy. This is a cat. |
| Beginner-2 | I like ___. I can ___. Do you ___? | I like apples. Can you jump? |
| Beginner-3 | There is/are ___. What/Where/When ___? | There are three books. What color is it? |
| Intermediate-1 | 现在进行时、一般过去时 | She is running. I went to the park. |
| Intermediate-2 | 比较级、条件句、被动语态 | The cheetah is faster than the rabbit. |

### 使用方法

1. 根据用户指定的目标年龄和难度，查表确定词汇和句型范围
2. 生成内容时控制在对应范围内
3. 如果用户指定了主题但主题词汇超出难度范围，做降级处理（用更简单的同义词）或标注提示

## 内容自检清单

生成完所有教学内容后、构建提示词前，逐项检查：

### 英语内容准确性
- [ ] 所有英文单词拼写正确？
- [ ] 所有英文句子语法正确？（主谓一致、时态、冠词、复数）
- [ ] 选择题答案是否唯一？干扰项是否合理？
- [ ] 填空题空格处的目标答案是否明确？
- [ ] 中文翻译是否准确？是否有歧义？
- [ ] 词汇难度是否匹配目标年龄？（查难度校准表）
- [ ] 词汇是否与主题直接相关？有无凑数词？

### 教学逻辑性
- [ ] 组件间词汇是否共享？（同一页的词汇卡和填空题应使用相同词汇集）
- [ ] 难度是否递进？（从词汇展示 → 配对练习 → 填空 → 选择，由易到难）
- [ ] 语法点讲解是否准确？例句是否用主题角色？
- [ ] 对话内容是否围绕本课知识点？
- [ ] 阅读理解的文章是否包含本课目标词汇？

### 数量合理性
- [ ] 词汇量是否在4-8个？
- [ ] 选择题数量是否在3-6道？
- [ ] 填空题数量是否在3-8道？
- [ ] 漫画格数是否在2-4格？
- [ ] 组件总数是否在3-6个？

## 组件间内容关联规则

同一张课件图上的多个组件不是独立的内容，它们应该共享同一套词汇和知识点。

### 核心规则：词汇集共享

每张课件图定义一个 **本课词汇集**（Vocab Set），所有组件使用这个集合的子集或全集。

```
示例：本课词汇集 = {Rabbit, Fox, Sloth, Cheetah, fast, cute}

Vocab Cards 展示全部6个词
Fill Blanks 的 word bank 包含这6个词
Multiple Choice 的选项从这6个词中选取
Matching 左列用前4个名词，右列用对应中文
```

### 组件内容流向

内容应按教学逻辑从输入到输出流动：

```
词汇展示（输入） → 配对/连线（识别） → 填空（应用） → 选择/判断（检测） → 造句/写作（输出）
```

不要把输出型组件（造句、写作）放在输入型组件（词汇卡）之前。

### 具体关联规则

| 组件A → 组件B | 关联规则 |
|---|---|
| Vocab Cards → Fill Blanks | Fill Blanks 的 word bank = Vocab Cards 的词汇集 |
| Vocab Cards → Matching | Matching 的左列 = Vocab Cards 中的名词子集 |
| Vocab Cards → Multiple Choice | MC 的选项从 Vocab Cards 词汇集中选取 |
| Grammar Points → Comic Dialogue | 漫画对话必须使用 Grammar Points 教的句型 |
| Grammar Points → Fill Blanks | 填空题的空格处 = Grammar Points 的语法结构 |
| Comic Dialogue → Role Play | Role Play 场景与漫画对话场景一致或延续 |
| Reading Comprehension → Writing | 写作题目与阅读文章主题关联 |
| Vocab Cards → Word Search | Word Search 中隐藏的单词 = Vocab Cards 词汇集 |
| Vocab Cards → Word Bingo | Bingo 卡上的单词 = Vocab Cards 词汇集 |
| Vocab Cards → Spelling | 拼写练习的单词 = Vocab Cards 词汇集子集 |

## 文字量与布局适配规则

生图模型对文字量的处理能力有限。文字太多 → 模糊、乱码、溢出。以下是经验规则。

### 单ZONE文字量上限

| 组件 | 单ZONE安全文字量 | 超限风险 |
|------|------------------|----------|
| Vocab Cards (4张) | 4个单词 + 4个翻译 = 8个词 | 低 |
| Vocab Cards (6张) | 6个单词 + 6个翻译 = 12个词 | 中 |
| Fill Blanks (6题) | 6个句子 + 6个word bank词 = 约60词 | 高 |
| Multiple Choice (4题) | 4题干 + 12选项 = 约50词 | 中 |
| Matching (4对) | 4词 + 4翻译 = 8个词 | 低 |
| Comic Dialogue (3格) | 3句对话 + 3翻译 = 约30词 | 中 |
| Grammar Points | 讲解1句 + 3例句 = 约40词 | 中 |
| Reading Comprehension | 50-80词短文 + 3题 = 约100词 | 极高 |
| Word Search (10×10) | 100个字母 + 6个单词 = 约106字符 | 极高 |
| True/False (4题) | 4个陈述句 = 约30词 | 低 |

### 文字量风险等级与对策

| 风险等级 | 说明 | 对策 |
|----------|------|------|
| 低 | 单ZONE文字少，模型能准确渲染 | 正常生成 |
| 中 | 文字量适中，需要清晰的布局描述 | 强调间距、字号、区域分隔 |
| 高 | 文字较多，模型可能模糊或溢出 | 减少题目数量，或拆成2个ZONE |
| 极高 | 文字极多，模型大概率乱码或丢字 | 拆成多张课件图，或减少组件 |

### 组件组合的文字量评估

生成提示词前，计算总文字量：

```
总文字量 = ZONE 1标题(约10词) + Σ(各组件ZONE文字量) + ZONE 3页脚(约5词)
```

| 总文字量 | 风险评估 | 建议 |
|----------|----------|------|
| < 50词 | 安全 | 3:4或16:9均可 |
| 50-100词 | 中等风险 | 建议3:4竖版，4K分辨率 |
| 100-150词 | 高风险 | 必须4K，减少组件数到3-4个 |
| > 150词 | 极高风险 | 拆成多张课件图 |

### 3:4竖版的组件容量参考

| 组件数 | 总文字量预估 | 可行性 |
|--------|-------------|--------|
| 2个（+Header+Footer） | 30-60词 | 安全，适合单一大组件 |
| 3个 | 50-90词 | 适中，推荐组合 |
| 4个 | 80-130词 | 较拥挤，需要精简文字 |
| 5个 | 120-180词 | 拥挤，不推荐 |
| 6个+ | >150词 | 极拥挤，拆页 |

## 失败诊断与修复策略

生图结果不理想时，按以下诊断树定位问题并修复。

### 问题1：文字乱码/不可读

```
诊断：
├─ 是否所有文字逐字写死在提示词中？
│  ├─ 否 → 修改：将所有文字内容逐字写入提示词
│  └─ 是 ↓
├─ 单ZONE文字量是否超过安全上限？
│  ├─ 是 → 修改：减少题目数量，或拆成多个ZONE
│  └─ 否 ↓
├─ 是否指定了4K分辨率？
│  ├─ 否 → 修改：添加"4K分辨率，文字清晰可读"
│  └─ 是 ↓
├─ 是否中英文混排在同一行？
│  ├─ 是 → 修改：中文和英文分行或分区域显示
│  └─ 否 ↓
└─ 尝试更换生图模型（参照模型适配策略）
```

### 问题2：文字溢出/超出区域

```
诊断：
├─ 组件数量是否过多（>4个）？
│  ├─ 是 → 修改：减少组件数量，或拆成多页
│  └─ 否 ↓
├─ ZONE区域是否描述了明确的高度比例？
│  ├─ 否 → 修改：为每个ZONE标注"占画面高度X%"
│  └─ 是 ↓
└─ 总文字量是否>100词？
   ├─ 是 → 修改：精简题干、减少选项数
   └─ 否 → 重试，或更换模型
```

### 问题3：布局混乱/组件重叠

```
诊断：
├─ 是否使用了ZONE系统描述每个组件位置？
│  ├─ 否 → 修改：用ZONE 2a/2b/2c...描述每个组件的位置和区域
│  └─ 是 ↓
├─ ZONE之间是否有明确分隔描述？
│  ├─ 否 → 修改：在ZONE之间添加"留白"或"分隔线"描述
│  └─ 是 ↓
└─ 组件排列方向是否明确？（横向/纵向/网格）
   ├─ 否 → 修改：明确描述"2×3网格"或"横向排列"
   └─ 是 → 重试
```

### 问题4：角色不像指定IP

```
诊断：
├─ 提示词中是否使用了角色名？
│  ├─ 否 → 修改：直接使用角色名（Woody, Buzz, Judy等）
│  └─ 是 ↓
├─ 角色外观描述是否足够详细？
│  ├─ 否 → 修改：添加颜色、服装细节、标志性特征
│  └─ 是 ↓
├─ 是否是不常见IP？模型可能不认识
│  ├─ 是 → 修改：web_search确认角色外观，用更详细的视觉描述替代角色名
│  └─ 否 ↓
└─ 在反向提示词中添加"no generic cartoon characters"迫使模型参考IP特征
```

### 问题5：风格不统一

```
诊断：
├─ 是否选定了风格预设？
│  ├─ 否 → 修改：从cute_cartoon/flat_design/hand_drawn中选一个
│  └─ 是 ↓
├─ 色彩系统是否在提示词末尾统一描述了？
│  ├─ 否 → 修改：添加"色彩：[色系名] — [具体颜色]"段落
│  └─ 是 ↓
└─ 不同ZONE的边框样式是否一致？
   ├─ 否 → 修改：统一为"全圆角"或"全直角"
   └─ 是 → 重试
```

## 生图模型适配策略

不同生图模型对文字渲染、复杂布局、中文支持的擅长程度不同。根据使用的模型调整提示词策略。

### 模型能力对比

| 模型 | 英文文字 | 中文文字 | 复杂布局 | 多组件 | 适合课件类型 |
|------|----------|----------|----------|--------|-------------|
| Gemini 系列 | 优秀 | 一般 | 良好 | 中等(≤4) | 词汇卡、选择题、判断题 |
| DALL-E 系列 | 良好 | 差 | 中等 | 少(≤3) | 简单布局、词汇卡 |
| 即梦/Seedream | 良好 | 优秀 | 良好 | 中等(≤5) | 含中文翻译的课件、复杂布局 |
| Midjourney | 差 | 极差 | 优秀 | 不推荐 | 不推荐用于课件（文字能力弱） |
| Nano Banana Pro | 优秀 | 良好 | 优秀 | 中等(≤4) | 高质量课件 |

### 适配策略

#### 使用 Gemini/DALL-E（英文文字能力强，中文弱）

1. 课件图中的可见文字以英文为主
2. 中文翻译改用拼音或去掉，在配套的MD文档中提供翻译
3. 提示词中强调 "all text in English only"
4. 减少每个ZONE的文字量到安全线以下
5. 组件数控制在3-4个以内

#### 使用即梦/Seedream（中文文字能力强）

1. 课件图中可以同时显示英文和中文翻译
2. 提示词用中文描述，可见文字中英文混合
3. 可以处理更复杂的布局（4-5个组件）
4. Word Search 等网格类组件优先用中文强的模型

#### 使用 Nano Banana Pro（文字和布局都强）

1. 可以处理最复杂的课件布局
2. 4K分辨率下文字清晰
3. 可以同时渲染中英文
4. 适合做高质量成品课件

### 通用适配规则

1. **提示词语言**：用户要求中文提示词就用中文，英文就用英文。不影响画面可见文字的语言。
2. **可见文字**：课件教学内容的语言由课程决定——英语课件就是英文为主，中文翻译辅助。
3. **模型不支持中文时**：去掉课件图中的中文翻译，单独在配套MD文档中列出。
4. **模型不支持复杂布局时**：减少组件数量，一页只放1-2个内容组件。

## 网格类组件生成算法

Word Search 和 Word Bingo 需要生成字母/单词网格。这不是生图模型能做的事——需要先用算法生成正确的网格内容，再把结果逐字写入提示词。

### Word Search 生成算法

```
输入：单词列表 words = ["Apple", "Banana", "Orange"]，网格大小 N = 10

步骤：
1. 初始化 N×N 空网格，所有格子填 "."
2. 对每个单词 word：
   a. 随机选择方向：horizontal(→) / vertical(↓) / diagonal(↘)
   b. 随机选择起点 (row, col)，确保单词能放下（不超出边界）
   c. 检查路径上的格子是否为空或与单词字母一致（允许交叉）
   d. 如果冲突，换一个起点重试（最多尝试20次）
   e. 放置单词
3. 用随机大写字母填充剩余的 "." 格子
4. 输出网格内容（逐行），用于写入提示词

示例输出：
A P P L E X K J F B
B X G R A P E K S X
A B A N A N A L B Z
N S T R A W B E R R
A G R A P E Z K R Y
...
```

### Word Bingo 生成算法

```
输入：词汇集 vocab = ["Rabbit", "Fox", "Sloth", "Cheetah", "fast", "cute", "jump", "run", "big"]，网格大小 3×3

步骤：
1. 从 vocab 中随机选取 8 个词（3×3 - 1个FREE格）
2. 打乱顺序
3. 填入 3×3 网格，中心格（第2行第2列）标 "FREE"
4. 输出网格内容

示例输出：
| Rabbit  | Fox     | Sloth   |
| Cheetah | FREE    | fast    |
| jump    | run     | cute    |
```

### 注意事项

1. **Word Search 的字母网格必须逐行写入提示词** — 不能只说"生成一个含这些单词的字母网格"，生图模型不会正确放置字母。必须把每一行的字母逐字写死。
2. **单词方向控制** — Beginner难度只放横向和纵向，不放对角线。Intermediate可以加对角线。
3. **网格大小与单词数量匹配** — 10×10网格最多放6-8个单词。8×8最多放4-5个。
4. **Bingo 网格的每个格子里是单词**，不是字母。

## 多页课件学习路径规划

做多页系列课件时（如8页寒暑假预习课件），需要系统性规划学习路径。

### 学习路径模型

```
第1-2页：词汇导入（新词输入）
第3-4页：句型/语法（结构学习）
第5-6页：综合练习（应用巩固）
第7页：游戏复习（趣味巩固）
第8页：阅读+自评（输出检测）
```

### 词汇分布规划

多页课件的词汇不应每页完全不同，也不应每页完全重复。

| 页码 | 词汇策略 | 示例 |
|------|----------|------|
| 第1页 | 引入核心词汇A组(4-6词) | Cowboy, Astronaut, Cowgirl, Dinosaur |
| 第2页 | 用A组词汇做练习（颜色/数字） | 复用A组词，加颜色词 Brown, Green |
| 第3页 | 引入动作词B组(4-6词) | Jump, Fly, Run, Walk |
| 第4页 | 用A+B组词汇做对话 | 复用A+B组词 |
| 第5页 | 引入语法点，用A+B组词汇造句 | 复用A+B组词 |
| 第6页 | 综合练习，A+B组词汇混合 | 复用全部词汇 |
| 第7页 | 游戏复习，A+B组词汇 | 复用全部词汇 |
| 第8页 | 阅读理解，文章包含A+B组词汇 | 复用全部词汇 |

### 难度递进规则

| 页码 | 难度 | 组件类型 | 教学目标 |
|------|------|----------|----------|
| 1-2 | 最易 | 展示型（词汇卡、图文配对） | 认读 |
| 3-4 | 易 | 互动型（漫画对话、角色扮演） | 理解 |
| 5-6 | 中 | 练习型（填空、选择、语法） | 应用 |
| 7 | 中 | 游戏型（搜索、Bingo） | 巩固 |
| 8 | 较难 | 输出型（阅读、写作、自评） | 检测 |

### 多页课件生成顺序

1. 先规划全部页面的词汇集和知识点分布
2. 确认每页的组件组合
3. 逐页生成教学内容（确保词汇复用和难度递进）
4. 逐页构建提示词
5. 逐页出图
6. 全部出完后检查一致性（角色外观、风格、色彩系统是否统一）
