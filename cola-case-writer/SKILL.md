# Cola Case Writer

把最近帮用户做的事情写成结构化的案例文档。

## 触发方式

用户说"写个案例"/"记录一下你做了什么"/"总结最近做的事"时调用。

## 输出格式

每个案例按以下结构输出：

```markdown
## 案例：{标题}

**用户需求：** 一句话描述用户想要什么
**完成状态：** ✅完成 / ⚠️部分完成 / ❌未完成

### 做了什么
简述最终交付的成果。

### 过程
1. 第一步做了什么
2. 第二步做了什么
3. ...

### 困难与踩坑
- 遇到了什么问题
- 怎么解决的（或没解决）

### 亮点
- 做得好的地方
- 用了什么巧妙的方法

### 使用的工具/Skill
- 列出用到的工具和Skill

### 反思
- 下次可以怎么做得更好
```

## 执行步骤

1. **回顾最近的对话和任务** — 从work items、memory、session history中提取最近完成的任务
2. **按任务逐个写案例** — 每个任务一个案例，按上面的格式
3. **合并成一篇文档** — 加上日期、用户信息（脱敏）、总览
4. **保存** — 输出为markdown文件，或写入飞书/OB

## 注意事项

- 如实记录，包括失败和踩坑，不要只写好的
- 困难部分要具体：什么问题、为什么发生、怎么处理的
- 亮点要有数据支撑：比如"143篇文章0失败"而不是"做得很好"
- 脱敏：不暴露用户的API key、密码、敏感个人信息
- 反思要真诚：下次可以改进什么
